# 💻 Windows Command Line

The Windows Command Prompt (`cmd.exe`) is Microsoft's default **Command-Line Interface (CLI)** for interacting with the operating system.

While most users are comfortable using a **Graphical User Interface (GUI)**, mastering the command line can dramatically improve your efficiency, especially when managing systems, troubleshooting issues, automating repetitive tasks, or administering remote servers.

Think of the Command Prompt as a direct way to communicate with Windows without navigating through menus and windows.

- Display basic system information

## 🧭 Task 2: Basic System Information

Before running commands, it's important to understand that Windows executes commands located within its **PATH** environment variable.

### 🌍 Viewing Environment Variables

Use the following command:

```cmd
set
```

Example output:

```text
Path=C:\Windows\system32;
C:\Windows;
C:\Windows\System32\WindowsPowerShell\v1.0\
...
```

The `Path` variable tells Windows where to search for executable programs.

![](https://github.com/Deeptig9138/CyberSecurity101---THM/blob/main/screenshots/M4/m4.2.png)

### 🖥️ Checking the Windows Version

To display the installed Windows version:

```cmd
ver
```

Example:

```text
Microsoft Windows [Version 10.0.17763.1821]
```

This is useful when identifying the operating system during troubleshooting or penetration testing.

![](https://github.com/Deeptig9138/CyberSecurity101---THM/blob/main/screenshots/M4/m4.3.png)

### 📋 Viewing Detailed System Information

For a comprehensive overview of the system, use:

```cmd
systeminfo
```

Example output:

```text
Host Name: WIN-SRV-2019
OS Name: Microsoft Windows Server 2019 Datacenter
OS Version: 10.0.17763
Processor(s): ...
Total Physical Memory: ...
```

This command provides information such as:

- Computer Name
- Windows Edition
- Build Number
- BIOS Version
- Installed RAM
- Processor
- Domain Information
- Boot Time
- Hotfixes

### 📖 Viewing Long Output

Some commands generate a large amount of output.

Instead of scrolling endlessly, pipe the output through **more**.

Example:

```cmd
driverquery | more
```

Benefits:

- Displays one page at a time
- Press **Space** for the next page
- Press **Enter** for one line
- Press **CTRL + C** to exit

### 🛠️ Helpful Commands

| Command | Purpose |
|----------|----------|
| `help` | Displays help for commands |
| `cls` | Clears the Command Prompt screen |
| `set` | Displays environment variables |
| `ver` | Displays Windows version |
| `systeminfo` | Displays detailed system information |

#### What is the OS version of the Windows VM?

```text

```

#### What is the hostname of the Windows VM?


## 🧭 Task 3: Network Troubleshooting

Networking commands are essential for both system administrators and penetration testers.

These commands help identify:

- IP addresses
- DNS servers
- Routing issues
- Active connections
- Open ports

### 🌐 Checking Network Configuration

Use:

```cmd
ipconfig
```

Example output:

```text
IPv4 Address : 10.10.230.237
Subnet Mask  : 255.255.0.0
Gateway      : 10.10.0.1
```

This quickly displays:

- IP Address
- Subnet Mask
- Default Gateway

![](https://github.com/Deeptig9138/CyberSecurity101---THM/blob/main/screenshots/M4/m4.4.png)

### 🔍 Viewing Complete Network Details

For more information:

```cmd
ipconfig /all
```

Additional details include:

- MAC Address
- DHCP Status
- DNS Servers
- Lease Information
- Physical Adapter Details
- IPv6 Addresses

This command is commonly used during network troubleshooting.

### 📡 Testing Connectivity

To check whether another system is reachable:

```cmd
ping example.com
```

Example output:

```text
Reply from 93.184.215.14
```

#### What Ping Tells You

- Whether the target is reachable
- Packet loss
- Network latency
- Round-trip time (RTT)

Example statistics:

```text
Packets Sent: 4
Received: 4
Lost: 0
Average: 78ms
```

![](https://github.com/Deeptig9138/CyberSecurity101---THM/blob/main/screenshots/M4/m4.5.png)

### 🛣️ Tracing Network Routes

Use:

```cmd
tracert example.com
```

Unlike `ping`, `tracert` shows every router (hop) between you and the destination.

Example:

```text
1 → Gateway
2 → ISP Router
3 → Backbone Router
...
15 → example.com
```

Useful when diagnosing:

- Routing problems
- Slow network paths
- Connectivity failures

### 🌍 Looking Up DNS Records

Use:

```cmd
nslookup example.com
```

Example:

```text
Name: example.com
Address: 93.184.215.14
```

![](https://github.com/Deeptig9138/CyberSecurity101---THM/blob/main/screenshots/M4/m4.6.png)

You can also specify a DNS server:

```cmd
nslookup example.com 1.1.1.1
```

Common public DNS servers:

| DNS Provider | Address |
|--------------|----------|
| Cloudflare | 1.1.1.1 |
| Google | 8.8.8.8 |

### 🔌 Viewing Active Connections

Use:

```cmd
netstat
```

Example:

```text
TCP
10.10.230.237:22
ESTABLISHED
```

This shows:

- Active network connections
- Listening ports
- Connection states

![](https://github.com/Deeptig9138/CyberSecurity101---THM/blob/main/screenshots/M4/m4.7.png)

### ⚙️ Useful Netstat Options

| Option | Description |
|----------|-------------|
| `-a` | Shows all listening ports and connections |
| `-b` | Shows executable associated with connection |
| `-o` | Displays Process ID (PID) |
| `-n` | Shows numerical addresses instead of hostnames |

Commonly used command:

```cmd
netstat -abon
```

Example output:

```text
TCP
0.0.0.0:22
LISTENING
2116
[sshd.exe]
```

This tells us:

- Port 22 is open
- SSH is running
- PID is 2116

### 🛠️ Common Networking Commands

| Command | Purpose |
|----------|----------|
| `ipconfig` | Display IP configuration |
| `ipconfig /all` | Display detailed network configuration |
| `ping` | Test connectivity |
| `tracert` | Trace network route |
| `nslookup` | Resolve DNS records |
| `netstat` | Display network connections |

### 💡 Pentester Tip

During reconnaissance, these commands are frequently used to identify:

- Network configuration
- Open ports
- Running services
- Reachability
- DNS resolution
- Internal routing paths

Understanding these commands is essential before moving on to more advanced enumeration techniques.

### ✅ Answers

#### Which command can we use to look up the server's physical address (MAC address)?

```text
ipconfig /all
```

```text

```

## 🧭 Task 4: File and Disk Management

One of the most common tasks performed in the Windows Command Prompt is managing files and directories. Whether you're navigating folders, creating backups, or organizing files, knowing these commands can significantly speed up your workflow.

### 📂 Working with Directories

#### 📍 Display the Current Directory

To see your current working directory, use:

```cmd
cd
```

Example:

```text
C:\Users\strategos
```

This is the CLI equivalent of asking:

> "Where am I?"

#### 📑 Listing Directory Contents

To display the contents of the current directory:

```cmd
dir
```

Example output:

```text
Directory of C:\Users\strategos

Desktop
Documents
Downloads
Pictures
Videos
```

This command lists:

- 📁 Folders
- 📄 Files
- 💾 Free disk space
- 📅 Creation dates

##### Useful `dir` Options

| Command | Description |
|----------|-------------|
| `dir` | Lists files and folders |
| `dir /a` | Shows hidden and system files |
| `dir /s` | Lists all files recursively, including subdirectories |

Example:

```cmd
dir /a
```

#### 🌳 Display Folder Structure

To display directories in a tree-like format:

```cmd
tree
```

Example:

```text
C:.
├── Desktop
├── Documents
├── Downloads
├── Pictures
└── Videos
```

This is useful when visualizing large directory structures.

#### 🚶 Navigating Directories

Move into a directory:

```cmd
cd Users
```

Move back one level:

```cmd
cd ..
```

Example:

```cmd
C:\> cd Users

C:\Users> cd ..

C:\>
```

#### 📁 Creating and Removing Directories

Create a folder:

```cmd
mkdir backup_files
```

Delete a folder:

```cmd
rmdir backup_files
```

Example:

```cmd
mkdir Reports
rmdir Reports
```

> ⚠️ `rmdir` only removes empty directories. Non-empty folders require additional options.

### 📄 Working with Files

#### 👀 Viewing File Contents

Display the contents of a text file:

```cmd
type notes.txt
```

Example output:

```text
This is a sample text file.
```

#### Viewing Long Files

For lengthy files, use:

```cmd
more notes.txt
```

Navigation:

- **Space** → Next page
- **Enter** → Next line
- **CTRL + C** → Exit

#### 📋 Copying Files

Copy a file:

```cmd
copy test.txt test2.txt
```

Example:

```cmd
copy report.txt backup.txt
```

This creates an identical copy of the file.

#### 🚚 Moving Files

Move a file to another directory:

```cmd
move report.txt ..
```

This moves the file one directory higher.

You can also rename a file while moving it:

```cmd
move report.txt C:\Backups\report_old.txt
```

#### 🗑️ Deleting Files

Delete a file:

```cmd
del report.txt
```

or

```cmd
erase report.txt
```

Both commands perform the same action.

> ⚠️ Deleted files are permanently removed—they do **not** go to the Recycle Bin.

#### ⭐ Using Wildcards

Wildcards allow you to work with multiple files simultaneously.

Example:

```cmd
copy *.md C:\Markdown
```

This copies **all Markdown (`.md`) files** to the destination folder.

Common wildcards:

| Wildcard | Meaning |
|----------|---------|
| `*` | Any number of characters |
| `?` | A single character |

Examples:

```cmd
*.txt
```

All text files.

```cmd
report?.txt
```

Matches:

```text
report1.txt
report2.txt
```

But not:

```text
report10.txt
```

### 📌 Common File Management Commands

| Command | Purpose |
|----------|----------|
| `cd` | Display or change directory |
| `dir` | List files and folders |
| `tree` | Display folder hierarchy |
| `mkdir` | Create a directory |
| `rmdir` | Remove a directory |
| `type` | Display file contents |
| `more` | View long files page-by-page |
| `copy` | Copy files |
| `move` | Move or rename files |
| `del` / `erase` | Delete files |

### 💡 Pentester Tip

During engagements, you'll frequently use these commands to:

- 📂 Browse user directories
- 📄 Read configuration files
- 📋 Copy evidence
- 🚚 Move payloads
- 🗑️ Remove temporary files
- 📁 Organize collected artifacts

Mastering these basic commands makes navigating compromised Windows systems much more efficient.

## 🧭 Task 5: Task and Process Management

Processes represent programs currently running on the system.

Windows Command Prompt provides several commands for viewing and managing these processes without opening Task Manager.

### 📊 Listing Running Processes

Display all running processes:

```cmd
tasklist
```

Example output:

```text
Image Name        PID
lsass.exe         592
svchost.exe       704
explorer.exe     3156
```

Information shown includes:

- Process Name
- Process ID (PID)
- Session
- Memory Usage

![](https://github.com/Deeptig9138/CyberSecurity101---THM/blob/main/screenshots/M4/m4.9.png)

### 🔍 Filtering Processes

The process list can become very long.

To filter results:

```cmd
tasklist /FI "imagename eq sshd.exe"
```

Example:

```text
sshd.exe
PID 2116
```

The `/FI` option applies a filter.

![](https://github.com/Deeptig9138/CyberSecurity101---THM/blob/main/screenshots/M4/m4.10.png)

Other useful examples:

```cmd
tasklist /FI "imagename eq notepad.exe"
```

```cmd
tasklist /FI "PID gt 2000"
```

### ❌ Terminating a Process

Kill a process using its Process ID (PID):

```cmd
taskkill /PID 1516
```

Example:

```cmd
taskkill /PID 4567
```

Windows immediately terminates the specified process.

> ⚠️ Be careful when terminating system processes, as doing so may destabilize or crash the operating system.

### 📌 Common Process Commands

| Command | Purpose |
|----------|----------|
| `tasklist` | List all running processes |
| `tasklist /FI` | Filter running processes |
| `taskkill /PID` | Terminate a process using its PID |

### 💡 Pentester Tip

Process enumeration is often performed after gaining access to a Windows system.

It helps identify:

- Running applications
- Security software (AV/EDR)
- Remote access tools
- Database services
- Interesting user applications

This information is valuable for privilege escalation, persistence, and post-exploitation.

### ✅ Answers

#### What command would you use to find the running processes related to `notepad.exe`?

```text
tasklist /FI "imagename eq notepad.exe"
```

#### What command can you use to kill the process with PID 1516?

```text
taskkill /PID 1516
```

### 💡 Helpful Reminder

Most Windows commands provide built-in documentation.

Simply append:

```cmd
/?
```

Example:

```cmd
ipconfig /?
```

or

```cmd
netstat /?
```

This displays usage information, options, and examples.

### 📖 Using `more`

The `more` command can be used in two different ways:

### View a Text File

```cmd
more file.txt
```

### View Long Command Output

```cmd
systeminfo | more
```

This allows you to read lengthy output one page at a time.

### ✅ Answers

#### What command can you use to restart a system?

```text
shutdown /r
```

#### What command can you use to abort a scheduled shutdown?

```text
shutdown /a
```

## 📝 Key Takeaways

- Windows Command Prompt (`cmd.exe`) is the default Windows command-line interpreter.
- `systeminfo` provides detailed information about the operating system and hardware.
- `ipconfig`, `ping`, `tracert`, `nslookup`, and `netstat` are essential networking commands.
- `cd`, `dir`, `tree`, `mkdir`, and `rmdir` help navigate and manage directories.
- `copy`, `move`, `type`, `more`, and `del` simplify file management.
- Wildcards (`*` and `?`) allow operations on multiple files.
- `tasklist` and `taskkill` are used to view and manage running processes.
- Most commands support the `/ ?` option to display built-in help.
- Command-line proficiency is essential for system administrators, penetration testers, SOC analysts, DFIR investigators, and cybersecurity professionals.
