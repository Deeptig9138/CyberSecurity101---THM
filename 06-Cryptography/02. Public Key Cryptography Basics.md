## 🧭 Task 2: Common Use of Asymmetric Encryption

Before discussing algorithms, let's understand the purpose of public key cryptography.

When communicating online, we want to ensure:

### 🔐 Authentication

Verifying that we are communicating with the correct person or server.

### ✅ Authenticity

Confirming that data genuinely comes from the claimed sender.

### 🛡 Integrity

Ensuring data has not been modified during transmission.

### 🔒 Confidentiality

Preventing unauthorized parties from reading the data.

### 📦 The Lock-and-Box Analogy

Imagine you want to send secret instructions to a friend.

Your friend gives you:

> A lock

Only they possess:

> The key to that lock

You place the instructions in a box, lock it, and send it.

Your friend unlocks the box using their key.

### 🔄 Cryptographic Equivalent

| Analogy | Cryptography |
|----------|-------------|
| Secret Code | Symmetric Cipher + Key |
| Lock | Public Key |
| Lock Key | Private Key |

### Why This Matters

Asymmetric encryption is slower than symmetric encryption.

Therefore:

1. Public key cryptography exchanges a secret key securely.
2. Symmetric encryption handles the actual communication.

This combines:

- Security
- Speed
- Scalability

### ✅ Answer

**In the analogy presented, what real object is analogous to the public key?**

`Lock`

## 🧭 Task 3: RSA

### 🔑 What is RSA?

RSA is one of the most widely used public key cryptosystems.

It enables:

- Secure communication
- Digital signatures
- Authentication

over insecure networks.

### 🧮 Why RSA Works

RSA security relies on a difficult mathematical problem:

> Factoring very large numbers.

### Easy Direction

Multiply two prime numbers:

```text
113 × 127 = 14351
```

Simple.

### Hard Direction

Given:

```text
14351
```

Find:

```text
113 × 127
```

Much harder.

### Real-World RSA

Modern RSA uses:

- Prime numbers hundreds of digits long
- Numbers too large to factor efficiently

### RSA Variables (Important for CTFs)

| Variable | Meaning |
|-----------|----------|
| p | Prime Number |
| q | Prime Number |
| n | p × q |
| ϕ(n) | (p−1)(q−1) |
| e | Public Exponent |
| d | Private Exponent |
| m | Plaintext |
| c | Ciphertext |

### Public & Private Keys

#### Public Key

```text
(n, e)
```

Used for encryption.

#### Private Key

```text
(n, d)
```

Used for decryption.

### 🏴 RSA in CTFs

Common tools include:

- RsaCtfTool
- rsatool

Challenges often provide:

```text
p
q
n
e
d
c
```

and require recovering the plaintext.

### ✅ Answers

**Knowing that p = 4391 and q = 6659. What is n?**

```text
n = p × q
n = 4391 × 6659
n = 29239669
```

Answer:

`29239669`

**Knowing that p = 4391 and q = 6659. What is ϕ(n)?**

```text
ϕ(n) = (p−1)(q−1)

= 4390 × 6658

= 29228620
```

Answer:

## 🧭 Task 4: Diffie-Hellman Key Exchange

One major challenge of symmetric encryption is:

> Securely sharing the secret key.

Diffie-Hellman solves this problem.

### 🔄 What is Diffie-Hellman?

Diffie-Hellman allows two parties to:

- Establish a shared secret
- Over an insecure network
- Without sending the secret itself

### 👥 Example Participants

Traditionally:

- Alice
- Bob

#### Public Values

```text
p = Prime Number
g = Generator
```

Known to everyone.

#### Private Values

```text
a = Alice's Secret
b = Bob's Secret
```

Never shared.

#### Public Values Generated

Alice:

```text
A = g^a mod p
```

Bob:

```text
B = g^b mod p
```

#### Shared Secret

Alice computes:

```text
B^a mod p
```

Bob computes:

```text
A^b mod p
```

Both arrive at:

```text
Same Secret Key
```

### Why It's Useful

Diffie-Hellman:

✅ Exchanges keys securely

RSA:

✅ Authenticates identities

Together they power:

- TLS
- HTTPS
- VPNs
- SSH

### ✅ Answers

**p = 29, g = 5, a = 12. What is A?**

```text
A = 5^12 mod 29
A = 7
```

Answer:

`7`

**p = 29, g = 5, b = 17. What is B?**

```text
B = 5^17 mod 29
B = 9
```

Answer:

`9`

**Key calculated by Bob**

```text
9^12 mod 29
```

Result:

`24`

**Key calculated by Alice**

```text
7^17 mod 29
```

Result:

`24`
```
Both derive the same shared secret.
```

## 🧭 Task 5: SSH

SSH uses public key cryptography extensively.

### 🖥 Authenticating the Server

When connecting to a server for the first time:

```bash
ssh user@host
```

You may see:

```text
The authenticity of host can't be established.
```

SSH asks whether you trust the server's public key fingerprint.

### Why?

To prevent:

> Man-in-the-Middle (MITM) attacks

A malicious server could pretend to be the real one.

### 👤 Authenticating the Client

After verifying the server:

The server must verify you.

Two common methods:

#### Password Authentication

```text
Username + Password
```

#### Key Authentication

```text
Public Key + Private Key
```

Recommended and more secure.

### 🔑 Generating SSH Keys

Example:

```bash
ssh-keygen -t ed25519
```

Supported algorithms:

- RSA
- DSA
- ECDSA
- ECDSA-SK
- Ed25519
- Ed25519-SK

### Common Algorithms

#### RSA

Widely supported and commonly used.

#### DSA

Designed specifically for signatures.

#### ECDSA

Uses elliptic curve cryptography.

#### Ed25519

Modern, fast, and secure.

#### Security Key Variants

- ECDSA-SK
- Ed25519-SK

Provide hardware-backed protection.

### ⚠️ Private Key Security

Treat private keys like passwords.

Never:

❌ Share them

Always:

✅ Protect them

Optional:

✅ Encrypt them using a passphrase

### SSH Files

#### Public Key

```text
id_ed25519.pub
```

Shared with servers.

#### Private Key

```text
id_ed25519
```

Kept secret.

### Authorized Keys

File:

```text
~/.ssh/authorized_keys
```

Contains public keys allowed to authenticate.

### SSH During Pentesting

Attackers often use SSH keys to:

- Upgrade shells
- Create persistence
- Maintain access

A public key added to:

```text
authorized_keys
```

can function as a stealthy backdoor.

## 🧭 Task 6: Digital Signatures and Certificates

### ✍️ What is a Digital Signature?

Digital signatures provide:

- Authenticity
- Integrity
- Non-repudiation

They prove:

- Who signed a file
- That it has not changed

### How It Works

Sender:

```text
Private Key
    ↓
 Sign
```

Receiver:

```text
Public Key
    ↓
 Verify
```

Only the owner of the private key could have created the signature.

### ⚠️ Digital Signature vs Image Signature

Not secure:

❌ Pasting a scanned signature image

Secure:

✅ Cryptographically signing the document

### 📜 Certificates

Certificates prove identity online.

Without certificates:

How would your browser know it is talking to the real website?

### HTTPS Example

When visiting a website:

```text
https://example.com
```

The server presents a certificate.

### Certificate Chain of Trust

```text
Root CA
    ↓
 Intermediate CA
    ↓
 Website Certificate
```

Browsers trust Root Certificate Authorities.

Examples:

- DigiCert
- GlobalSign
- Sectigo
- Let's Encrypt

### TLS Certificates

Required for:

```text
HTTPS
```

They provide:

- Encryption
- Authentication
- Trust

### Let's Encrypt

Provides:

> Free TLS certificates

for website owners.

### ✅ Answers

**What does a remote web server use to prove itself to the client?**

`Certificate`

**What would you use to get a free TLS certificate for your website?**

`Let's Encrypt`

## 🧭 Task 7: PGP and GPG

### 📧 What is PGP?

PGP stands for:

> Pretty Good Privacy

Used for:

- Email encryption
- File encryption
- Digital signatures

### GPG

GPG (GnuPG) is:

> Open-source implementation of OpenPGP

### Common Uses

#### Encrypt Files

Protect sensitive documents.

#### Encrypt Emails

Ensure confidentiality.

#### Sign Messages

Verify authenticity and integrity.

### Generating GPG Keys

Example:

```bash
gpg --full-gen-key
```

During setup:

- Choose algorithm
- Set expiration date
- Provide name
- Provide email

### Importing Keys

```bash
gpg --import backup.key
```

### Decrypting Files

```bash
gpg --decrypt secret.gpg
```

### GPG Key Protection

Like SSH keys:

- Public Key → Shared
- Private Key → Secret

Private keys may be protected with passphrases.

Tools such as:

- John the Ripper
- gpg2john

can attempt passphrase cracking.
