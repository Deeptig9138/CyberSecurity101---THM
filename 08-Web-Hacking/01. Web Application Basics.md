This room introduces the fundamental concepts behind **Web Applications**, including URLs, HTTP requests and responses, HTTP methods, status codes, headers, and common web security mechanisms.

## 🧭 Task 2: Web Application Overview

A web application can be compared to a planet.

Users explore the visible surface through a web browser, while many unseen systems operate beneath the surface to keep everything running.

### 🎨 Front End

The Front End is everything users see and interact with in a web browser.

Technologies commonly used include:

* HTML
* CSS
* JavaScript

#### HTML (HyperText Markup Language)

HTML provides the structure and content of a webpage.

Examples:

* Text
* Images
* Buttons
* Forms
* Links

Think of HTML as the blueprint or skeleton of a webpage.

#### CSS (Cascading Style Sheets)

CSS controls:

* Colours
* Fonts
* Layouts
* Spacing
* Animations

Without CSS, webpages would look plain and unstyled.

#### JavaScript (JS)

JavaScript adds functionality and interactivity.

Examples:

* Dropdown menus
* Form validation
* Dynamic content updates
* Interactive dashboards

JavaScript allows webpages to make decisions and respond to user actions.

### ⚙️ Back End

The Back End consists of components users do not directly see.

These systems process requests, store data, and deliver content.

#### 🗄 Database

Databases store and retrieve information.

Examples:

* User accounts
* Password hashes
* Preferences
* Application data

Popular databases include:

* MySQL
* PostgreSQL
* MongoDB
* Microsoft SQL Server

#### 🖥 Web Server & Infrastructure

Infrastructure components may include:

* Web Servers
* Application Servers
* Storage Systems
* Networking Devices
* Supporting Software

These components enable web applications to function properly.

#### 🛡 Web Application Firewall (WAF)

A WAF filters incoming traffic before requests reach the web server.

Benefits:

* Blocks malicious requests
* Helps mitigate attacks
* Adds an additional security layer

Examples of attacks a WAF may help prevent:

* SQL Injection
* Cross-Site Scripting (XSS)
* Malicious bots

### ✅ Answers

**Which component on a computer is responsible for hosting and delivering content for web applications?**

`web server`

**Which tool is used to access and interact with web applications?**

`web browser`

**Which component acts as a protective layer, filtering incoming traffic to block malicious attacks and ensuring the security of the web application?**

`web application firewall`

## 🧭 Task 3: Uniform Resource Locator (URL)

A URL (Uniform Resource Locator) is the address used to locate resources on the internet.

Examples:

```text
https://tryhackme.com
https://example.com/login
https://youtube.com/watch?v=123
```

### 🔍 Anatomy of a URL

Example:

```text
https://user@example.com:443/blog/article?id=10#comments
```

#### Scheme

Defines the protocol used.

Examples:

```text
HTTP
HTTPS
```

HTTPS encrypts communication between the browser and server.

#### User

May contain authentication information.

Example:

```text
user@example.com
```

Rarely used today due to security concerns.

#### Host / Domain

Identifies the website.

Example:

```text
tryhackme.com
```

Attackers often abuse similar-looking domains through:

> Typosquatting

Examples:

```text
g00gle.com
faceboook.com
```

#### Port

Determines which service should receive the request.

Common ports:

| Protocol | Port |
| -------- | ---- |
| HTTP     | 80   |
| HTTPS    | 443  |

#### Path

Identifies the resource on the server.

Example:

```text
/api/users/123
```

#### Query String

Provides additional information.

Example:

```text
?id=123&sort=name
```

Query strings begin with:

```text
?
```

#### Fragment

Points to a specific section of a webpage.

Example:

```text
#comments
```

Fragments begin with:

```text
#
```

### ✅ Answers

**Which protocol provides encrypted communication between a web browser and a web server?**

`HTTPS`

**What term describes registering misspelled variations of popular domains?**

`Typosquatting`

**What part of a URL passes additional information such as search terms or form inputs?**

`Query String`

## 🧭 Task 4: HTTP Messages

HTTP messages are exchanged between:

* Client (Browser)
* Server

There are two types:

### HTTP Request

Sent from:

```text
Client → Server
```

Requests information or performs an action.

### HTTP Response

Sent from:

```text
Server → Client
```

Returns data or status information.

### Components of an HTTP Message

#### Start Line

Identifies the type of request or response.

#### Headers

Key-value pairs containing metadata.

#### Empty Line

Separates headers from the body.

#### Body

Contains actual data being transmitted.

### Why HTTP Messages Matter

Understanding HTTP helps with:

* Troubleshooting
* Web development
* Penetration testing
* Security analysis

### ✅ Answers

**Which HTTP message is returned by the web server after processing a client's request?**

`HTTP response`

**What follows the headers in an HTTP message?**

`empty line`

## 🧭 Task 5: HTTP Requests — Request Line & Methods

HTTP requests tell a web server what action should be performed.

### Request Line

Format:

```text
METHOD /path HTTP/version
```

Example:

```text
GET /login HTTP/1.1
```

### Common HTTP Methods

#### GET

Retrieves information.

Example:

```http
GET /users
```

#### POST

Sends data to create or update resources.

Example:

```http
POST /login
```

#### PUT

Replaces or updates existing resources.

Example:

```http
PUT /user/5
```

#### DELETE

Removes resources.

Example:

```http
DELETE /user/5
```

### Additional Methods

#### PATCH

Updates part of a resource.

#### HEAD

Returns headers only.

#### OPTIONS

Shows supported HTTP methods.

#### TRACE

Used for diagnostics and debugging.

#### CONNECT

Creates secure tunnels, often used with HTTPS.

### URL Path

The URL path identifies the target resource.

Example:

```text
https://tryhackme.com/api/users/123
```

Path:

```text
/api/users/123
```

### HTTP Versions

#### HTTP/0.9

Only supported GET requests.

#### HTTP/1.0

Introduced headers.

#### HTTP/1.1

Added:

* Persistent connections
* Chunked transfer encoding
* Improved caching

#### HTTP/2

Added:

* Multiplexing
* Header compression
* Prioritization

#### HTTP/3

Built on QUIC for faster and more secure communication.

### ✅ Answers

**Which HTTP version introduced persistent connections and chunked transfer encoding and remains widely used today?**

`HTTP/1.1`

**Which HTTP method describes communication options supported by a resource?**

`OPTIONS`

**Which component identifies the resource being requested on the web server?**

`URL Path`

## 🧭 Task 6: HTTP Requests — Headers & Body

### Request Headers

Headers provide additional information about requests.

### Common Request Headers

| Header       | Purpose                    |
| ------------ | -------------------------- |
| Host         | Target server              |
| User-Agent   | Browser information        |
| Referer      | Source URL                 |
| Cookie       | Stored session information |
| Content-Type | Data format                |

#### Host

```http
Host: tryhackme.com
```

Specifies the destination web server.

#### User-Agent

```http
User-Agent: Mozilla/5.0
```

Identifies the client software.

#### Referer

```http
Referer: https://google.com
```

Shows where the request originated.

#### Cookie

```http
Cookie: sessionid=12345
```

Contains stored client information.

#### Content-Type

```http
Content-Type: application/json
```

Describes the format of transmitted data.

### Request Body

Used primarily with:

* POST
* PUT
* PATCH

#### URL Encoded

```text
application/x-www-form-urlencoded
```

Example:

```text
name=John&country=US
```

#### Form Data

```text
multipart/form-data
```

Commonly used for:

* File uploads
* Images

#### JSON

```json
{
  "name": "John",
  "country": "US"
}
```

#### XML

```xml
<user>
  <name>John</name>
</user>
```

### ✅ Answers

**Which request header specifies the destination domain?**

`Host`

**What is the default content type for standard form submissions?**

`application/x-www-form-urlencoded`

**Which part of an HTTP request contains information like Host, User-Agent, and Content-Type?**

`Request Headers`

## 🧭 Task 7: HTTP Responses — Status Line & Status Codes

HTTP responses tell the client whether a request succeeded or failed.

### Status Line

Contains:

1. HTTP Version
2. Status Code
3. Reason Phrase

Example:

```http
HTTP/1.1 200 OK
```

### Status Code Categories

#### 1xx — Informational

Request received, continue processing.

Example:

```text
100 Continue
```

#### 2xx — Success

Request processed successfully.

Example:

```text
200 OK
```

#### 3xx — Redirection

Resource moved elsewhere.

Example:

```text
301 Moved Permanently
```

#### 4xx — Client Errors

Problem with the request.

Example:

```text
404 Not Found
```

#### 5xx — Server Errors

Problem occurred on the server.

Example:

```text
500 Internal Server Error
```

### Common Status Codes

| Code | Meaning               |
| ---- | --------------------- |
| 100  | Continue              |
| 200  | OK                    |
| 301  | Moved Permanently     |
| 404  | Not Found             |
| 500  | Internal Server Error |

### ✅ Answers

**What part of an HTTP response contains the version, status code, and reason phrase?**

`Status Line`

**Which category indicates internal server problems?**

`Server Error Responses`

**Which status code indicates that a resource could not be found?**

`404`

## 🧭 Task 8: HTTP Responses — Headers & Body

### Response Headers

Response headers provide metadata about the response.

#### Date

```http
Date: Fri, 23 Aug 2024 10:43:21 GMT
```

Indicates when the response was generated.

#### Content-Type

```http
Content-Type: text/html
```

Specifies the type of returned content.

#### Server

```http
Server: nginx
```

Identifies server software.

#### Set-Cookie

```http
Set-Cookie: sessionId=abc123
```

Stores cookies on the client.

Recommended flags:

```text
HttpOnly
Secure
```

#### Cache-Control

```http
Cache-Control: max-age=600
```

Controls caching behavior.

#### Location

```http
Location: /login
```

Used during redirections.

### Response Body

Contains the actual content returned.

Examples:

* HTML
* JSON
* Images
* XML
* Files

Always sanitize user input before displaying it to prevent:

* XSS
* Injection attacks

### ✅ Answers

**Which response header can reveal server software information?**

`Server`

**Which cookie flag ensures cookies are only transmitted over HTTPS?**

`Secure`

**Which cookie flag prevents JavaScript from accessing cookies?**

`HttpOnly`

## 🧭 Task 9: Security Headers

Security headers provide additional protection against common web attacks.

### 🛡 Content-Security-Policy (CSP)

Restricts where content can be loaded from.

Example:

```http
Content-Security-Policy:
default-src 'self';
script-src 'self' https://cdn.tryhackme.com;
style-src 'self';
```

### Common Directives

#### default-src

Default allowed sources.

#### script-src

Allowed JavaScript sources.

#### style-src

Allowed CSS sources.

### 🔒 Strict-Transport-Security (HSTS)

Forces browsers to use HTTPS.

Example:

```http
Strict-Transport-Security:
max-age=63072000;
includeSubDomains;
preload
```

### Common Directives

#### max-age

Duration of policy.

#### includeSubDomains

Apply policy to subdomains.

#### preload

Allows inclusion in browser preload lists.

### 🚫 X-Content-Type-Options

Example:

```http
X-Content-Type-Options: nosniff
```

Prevents browsers from guessing MIME types.

### 🔍 Referrer-Policy

Controls referrer information sharing.

Examples:

```http
Referrer-Policy: no-referrer
Referrer-Policy: same-origin
Referrer-Policy: strict-origin
Referrer-Policy: strict-origin-when-cross-origin
```

### Common Directives

#### no-referrer

Sends no referrer data.

#### same-origin

Only sends referrer data to same-origin sites.

#### strict-origin

Only sends origin information over equally secure protocols.

#### strict-origin-when-cross-origin

Provides more information for same-origin requests while restricting cross-origin requests.

### ✅ Answers

**Which CSP directive specifies where scripts may be loaded from?**

`script-src`

**Which HSTS directive applies HTTPS enforcement to subdomains?**

`includeSubDomains`

**Which X-Content-Type-Options directive prevents MIME type sniffing?**

`nosniff`
