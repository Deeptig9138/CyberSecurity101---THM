## 🧭 Task 2: DNS: Remembering Addresses

Computers, routers, and firewalls communicate strictly via numerical IP addresses. However, because humans cannot efficiently memorize thousands of network addresses, the internet implements the **Domain Name System (DNS)**. DNS maps human-readable domain strings (e.g., `example.com`) directly to target destination logical IP vectors.

DNS operates natively at the Application Layer (Layer 7 of the OSI Model). It relies primarily on **UDP Port 53** to facilitate fast queries, dropping back to **TCP Port 53** as an internal fallback mechanism when handling massive zones or structural data transfers.

### 🗃️ Foundational DNS Resource Record Taxonomy

Defenders must recognize the four core types of records that populate standard zone lookup pools:

* **A Record:** The standard address identifier. It binds a target hostname string directly to one or more logical **IPv4** addresses (e.g., resolving `example.com` to `93.184.215.14`).
* **AAAA Record (Quad-A):** Maps a hostname target explicitly to an **IPv6** address string (e.g., `2606:2800:21f:cb07:6820:80da:af6b:8b2c`).
* **CNAME Record (Canonical Name):** Creates an alias chain by mapping one domain name straight to another domain target (e.g., defining `www.example.com` as an alias that points to the canonical root `example.com`).
* **MX Record (Mail Exchange):** Specifies the distinct mail server assets responsible for handling inbound email routing paths on behalf of that entire domain structure.

### 💻 Command-Line Name Space Resolution

To manually audit a domain's records, analysts use the `nslookup` command-line tool. A standard query initiates a brief sequence of background requests to resolve the target domain:

```bash
user@TryHackMe$ nslookup www.example.com
Server:         127.0.0.53
Address:        127.0.0.53#53

Non-authoritative answer:
Name:   www.example.com
Address: 93.184.215.14
Name:   www.example.com
Address: 2606:2800:21f:cb07:6820:80da:af6b:8b2c
```

### ✅ Answers

#### Which DNS record type refers to IPv6?

```text
AAAA
```

#### Which DNS record type refers to the email server?

```text
MX
```

## 🧭 Task 3: WHOIS

For DNS records to function reliably across the global web, there must be a mechanism to assign administrative authority over domain structures. This authority is established when an entity formally purchases and signs up a domain name through an authorized domain registrar.

### 📜 Public Registration Data Trails

When registering a domain, the purchaser is legally required to provide accurate point-of-contact details. This information is saved publicly inside the **WHOIS** database repository.

A standard WHOIS search provides critical investigative metadata about a domain's ownership timeline, including the registration date, recent structural adjustments, expiration dates, target registrar abuse contacts, and the owner's operational details.

To protect internal personnel from public exposure, organizations often use privacy services (such as *Domains By Proxy*). These services substitute the registrant's personal names, phone numbers, and physical addresses with generic proxy placeholders within the public WHOIS directory.

```text
Domain Name: [REDACTED].COM
Registrar WHOIS Server: whois.godaddy.com
Updated Date: 2017-07-05T16:02:43Z
Creation Date: 1993-04-02T00:00:00Z
Registrar Registration Expiration Date: 2026-10-20T14:56:17Z
Registrant Name: Registration Private
Registrant Organization: Domains By Proxy, LLC
```

## 🧭 Task 4: HTTP(S): Accessing the Web

The core architecture of the web relies on the **Hypertext Transfer Protocol (HTTP)** and its encrypted counterpart, **HTTPS (Secure)**. Operating at the transport layer over **TCP Port 80** (HTTP) and **TCP Port 443** (HTTPS), this protocol establishes a structured request-response communication framework between a client's web browser and a remote web server.

### 🛠️ Core HTTP Request Methods

* `GET`: Requests and retrieves a specific resource (such as an HTML document, text file, or image asset) from the remote server.
* `POST`: Submits new data payloads to the server, commonly used when filling out web forms or uploading media files.
* `PUT`: Creates a new target resource or overwrites an existing file location on the server with updated data.
* `DELETE`: Requests that the server permanently purge a specified resource or asset path.

### 🔬 Auditing Web Headers via Raw TCP Sockets

When a web browser requests a page, the underlying network conversation involves exchanging several structural headers that are hidden from the end user. Analysts can review these raw exchanges by manually creating a TCP connection to port 80 using the `telnet` utility and typing structural HTTP commands directly into the socket:

```bash
user@TryHackMe$ telnet MACHINE_IP 80
Trying MACHINE_IP...
Connected to MACHINE_IP.
GET /flag.html HTTP/1.1
Host: telnet.thm
```

*(Pressing **Enter** twice transmits a trailing blank line, which signals the web server to process the request header and return the targeted document data block).*

### ✅ Answer

#### Use telnet to access the file flag.html on MACHINE_IP. What is the hidden flag?

```text
THM{TELNET-HTTP}
```

## 🧭 Task 5: FTP: Transferring Files

While HTTP is optimized for web browser rendering, the **File Transfer Protocol (FTP)** is designed specifically to move bulk files efficiently between local clients and remote servers. Because it features minimal application protocol overhead, FTP can achieve significantly faster raw transmission speeds than HTTP under identical network conditions.

### ⚙️ Command Channel Execution Blueprint

FTP operates using a **dual-channel architecture**. It uses a persistent command channel over **TCP Port 21** to handle authentication and state management, while spinning up separate, dynamic TCP connections to transfer directory listings and file data.

* `USER`: Submits the account authentication username string to the server interface.
* `PASS`: Submits the account password string to finalize access validation.
* `LIST`: Requests the server to return an inventory of files sitting inside the current directory path (triggered when typing `ls` in an FTP client).
* `RETR` (Retrieve): Downloads a target file from the remote storage server to the local client machine.
* `STOR` (Store): Uploads a local file payload from the client machine up to the remote storage server.

### 📥 Sample Anonymous Session Execution

Many public servers allow unauthenticated access via an **anonymous FTP profile**. This allows users to connect by entering `anonymous` as the username and leaving the password blank:

```text
user@TryHackMe$ ftp 10.48.176.240

226 Directory send OK.
ftp> type ascii
200 Switching to ASCII mode.
ftp> get flag.txt

ftp> quit
```

## 🧭 Task 6: SMTP: Sending Email

To transmit message traffic across mail architectures, networks rely on the **Simple Mail Transfer Protocol (SMTP)**. Operating over **TCP Port 25** by default, SMTP defines the exact rules governing how a mail client submits messages to a local mail server, and how mail servers relay those messages across the internet.

### ✉️ Functional SMTP Command Sequence

Sending an email via SMTP mimics the process of dropping a physical letter off at a local post office. The client greets the server, provides verified sender and recipient addresses, and attaches the payload data block:

* `HELO` / `EHLO`: Initiates the SMTP session conversation and prompts the server to identify itself.
* `MAIL FROM:`: Declares the originating email sender address profile (e.g., `<user@client.thm>`).
* `RCPT TO:`: Declares the target recipient mailbox address tracking the message destination.
* `DATA`: Notifies the server that the following lines contain the actual message contents (including the Subject header and body text).
* `.` (Single Period): Entering a single period on a line by itself signals the end of the email data block, prompting the server to accept and queue the message.
* `QUIT`: Terminates the connection session with the remote mail exchange.

```bash
user@TryHackMe$ telnet 10.48.176.240 25
220 example.thm ESMTP Exim 4.95 Ubuntu
HELO client.thm
250 example.thm Hello client.thm
MAIL FROM: <user@client.thm>
250 OK
RCPT TO: <strategos@server.thm>
250 Accepted
DATA
354 Enter message, ending with "." on a line by itself
Subject: Telnet email

Hello from telnet!
.
250 OK id=1sMrpq-0001Ah-UT
QUIT
```

### ✅ Answers

#### Which SMTP command indicates that the client will start the contents of the email message?

```text
DATA
```

#### What does the email client send to indicate that the email message has been fully entered?

```text
.
```

## 🧭 Task 7: POP3: Receiving Email

While SMTP handles outbound message transmissions, users need specialized protocols to connect to their mailboxes and retrieve messages. The **Post Office Protocol version 3 (POP3)** is a minimalist protocol designed to download emails from a remote mail server to a local client. It listens natively on **TCP Port 110**.

### 🗑️ Standard POP3 Operational Commands

POP3 works similarly to a physical mailbox: the client authentication profile logs in, downloads the new messages to local storage, and deletes the copies from the remote server by default to save storage space.

* `USER` / `PASS`: Provides the account credentials needed to access the target mailbox profile.
* `STAT`: Requests the server to return the total number of unread messages and their overall data size in octets.
* `LIST`: Enumerates the indexed message IDs alongside their respective byte sizes.
* `RETR [Message Number]`: Downloads the full content headers and body payload of a specified message index.
* `DELE [Message Number]`: Marks a targeted message index to be permanently deleted from the server once the session ends.
* `QUIT`: Finalizes all pending deletions and logs out of the active POP3 session.

```bash
user@TryHackMe$ telnet 10.48.176.240 110
+OK Dovecot ready.
USER linda
+OK
PASS Pa$$123
+OK Logged in.
LIST
1 407
4 512
.
RETR 4
[Message Content Renders Here]
.
QUIT
```

> ⚠️ **Cleartext Security Warning:** Standard POP3 transmissions do not use native encryption. As a result, anyone capturing network traffic with a tool like Wireshark can easily read authentication passwords and email content in plain text.

#### Looking at the traffic exchange, what is the name of the POP3 server running on the remote server?

## 🧭 Task 8: IMAP: Synchronizing Email

POP3 works well if you manage your mailbox from a single dedicated device. However, if you need to access your email from multiple devices simultaneously (such as an office workstation, a personal laptop, and a smartphone), POP3's download-and-delete model falls short. The **Internet Message Access Protocol (IMAP)** addresses this by synchronizing your entire mailbox across multiple active clients. It listens natively on **TCP Port 143**.

### 🔄 Multi-Device Mail Synchronization Mechanics

Unlike POP3, which deletes messages after they are downloaded, IMAP keeps your emails stored centrally on the mail server. It continuously synchronizes message statuses (such as whether an email has been read, flagged, moved into a folder, or deleted) across all connected client applications. This requires more storage space on the server but ensures a consistent mailbox state across all your devices.

The IMAP protocol uses structured command prefixes (labels like `A`, `B`, `C`) to handle complex queries:

* `LOGIN [Username] [Password]`: Authenticates the user profile to open the session connection.
* `SELECT [Mailbox]`: Chooses a specific folder target (such as the `inbox`) to interact with.
* `FETCH [Message Number] body[]`: Downloads the exact header maps and body contents of a targeted message index without removing it from the server.
* `MOVE` / `COPY`: Manages folder arrangements by moving or replicating message indexes between server directories.

```bash
user@TryHackMe$ telnet 10.48.176.240 143
* OK Dovecot ready.
A LOGIN strategos password123
A OK Logged in
B SELECT inbox
* 4 EXISTS
B OK [READ-WRITE] Select completed.
C FETCH 4 body[]
* 4 FETCH (BODY[] {445} ... )
C OK Fetch completed.
D LOGOUT
```

### ✅ Answer

#### What IMAP command retrieves the fourth email message?

```text
FETCH 4 body[]
```

## 🧭 Task 9: Conclusion

Mastering text-based Application Layer protocols is essential for understanding how network services function beneath their graphical user interfaces. Whether resolving domains via DNS, fetching web pages using HTTP, transferring files through FTP, or managing mailboxes via SMTP, POP3, and IMAP, these core standards form the foundation of modern digital communications.

### 📊 Master Protocol Port Reference Matrix

| Application Protocol | Transport Mechanism | Default Port Number | Core Protocol Purpose |
| --- | --- | --- | --- |
| **TELNET** | TCP | 23 | Unencrypted Remote Terminal Shell Access |
| **DNS** | UDP / TCP | 53 | Hostname-to-IP Logical Address Mapping |
| **HTTP** | TCP | 80 | Hypertext Transfer Protocol Web Delivery |
| **HTTPS** | TCP | 443 | Secure, Encrypted Web Traffic Delivery |
| **FTP** | TCP | 21 | Optimized File Storage and Transmission |
| **SMTP** | TCP | 25 | Outbound Email Routing and Relaying |
| **POP3** | TCP | 110 | Single-Device Mailbox Download & Purge |
| **IMAP** | TCP | 143 | Multi-Device Server-Side Mailbox Synchronization |

---

## 📝 Key Takeaways

* **Logical Name Mapping:** DNS acts as the phone book of the web, converting text hostnames into routing IP addresses using specialized record types like `A` (IPv4) and `AAAA` (IPv6).
* **Public Information Trails:** WHOIS records maintain publicly accessible registration profiles, providing key details on domain registration timelines and organizational targets.
* **Text-Driven Sockets:** Legacy protocols use plain-text commands that can be directly tested and debugged using tool extensions like Telnet.
* **Data Transmission Modes:** FTP uses independent network connections for its command and data channels to optimize file transmission speeds.
* **Email Protocol Demarcation:** Outbound emails depend entirely on SMTP loops, whereas inbound message retrieval relies on either POP3 (local download and delete) or IMAP (server-side cross-device synchronization).
