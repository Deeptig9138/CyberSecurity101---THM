# 🕷️ Burp Suite: The Basics

Burp Suite is one of the most widely used web application security testing frameworks in the cybersecurity industry. It enables security professionals to intercept, inspect, modify, and analyze HTTP/HTTPS traffic between a client and a server.

Burp is commonly used for:

* Web Application Penetration Testing
* API Security Testing
* Mobile Application Testing
* Vulnerability Assessment
* Bug Bounty Hunting
* Security Research

From a cybersecurity perspective, Burp Suite provides complete visibility into web traffic, allowing testers to identify vulnerabilities such as XSS, SQL Injection, Authentication Flaws, Access Control Issues, and API Misconfigurations.

## 🧭 Task 2: What is Burp Suite?

Burp Suite is a Java-based web application penetration testing framework developed by PortSwigger.

It is considered the industry standard tool for testing:

* Websites
* APIs
* Mobile Applications
* Web Services
  
### 🌐 What Does Burp Do?

Burp sits between your browser and a target web server.

It can:

* Capture Requests
* Capture Responses
* Modify Traffic
* Replay Requests
* Analyze Applications

This process is known as:

```text
Interception
```

### 📦 Burp Suite Editions

#### Community Edition

Free version designed for learning and manual testing.

Features include:

* Proxy
* Repeater
* Decoder
* Comparer
* Sequencer
* Limited Intruder

#### Professional Edition

Commercial version with advanced capabilities.

Additional features:

* Automated Vulnerability Scanner
* Unlimited Intruder
* Project Saving
* Report Generation
* Burp Collaborator
* Extension Support
* API Integration

#### Enterprise Edition

Server-based edition designed for continuous scanning.

Instead of manual testing, it continuously scans web applications for vulnerabilities.

Think of it as:

```text
Nessus for Web Applications
```


## 🧭 Task 3: Features of Burp Community

Even though Community Edition has limitations, it still contains powerful tools.

### 🕸️ Proxy

Intercepts and modifies traffic between the browser and target.

Common uses:

* View Requests
* Modify Parameters
* Bypass Validation
* Analyze Sessions

### 🔁 Repeater

Allows repeated testing of a single request.

Useful for:

* SQL Injection
* XSS
* Authentication Testing
* Parameter Manipulation

### 🚀 Intruder

Automates large numbers of requests.

Common uses:

* Brute Force Attacks
* Fuzzing
* Wordlist Testing

### 🔓 Decoder

Converts data between formats.

Examples:

```text
URL Encoding
Base64
Hex
HTML Encoding
```

### ⚖️ Comparer

Compares:

* Requests
* Responses
* Files
* Tokens

Comparison modes:

* Word Level
* Byte Level

### 🎲 Sequencer

Tests randomness of tokens such as:

* Session Cookies
* CSRF Tokens
* Password Reset Tokens

Poor randomness can lead to account compromise.

### 🔌 Extensions

Burp supports custom extensions written in:

* Java
* Python (Jython)
* Ruby (JRuby)

Extensions are managed through:

```text
BApp Store
```

Popular extension:

```text
Logger++
```

### ✅ Answers

**Which Burp Suite feature allows us to intercept requests between ourselves and the target?**

```text
Proxy
```

**Which Burp tool would we use to brute-force a login form?**

```text
Intruder
```

## 🧭 Task 8: Introduction to the Burp Proxy

The Proxy is the heart of Burp Suite.

### 🔄 Request Interception

When enabled:

```text
Intercept is on
```

Requests stop before reaching the server.

You can then:

* Edit
* Forward
* Drop
* Send to Other Tools

### 📜 HTTP History

Burp records all traffic automatically.

View captured traffic under:

```text
HTTP History
```

### 🔌 WebSocket Support

Burp can also intercept:

```text
WebSocket Traffic
```

Useful for modern web applications.

### ✏️ Match and Replace

Automatically modify requests and responses.

Examples:

* Change User-Agent
* Modify Cookies
* Rewrite Headers

### 📝 Key Takeaways

* Interception gives complete control over traffic.
* History logs all captured requests.
* Match & Replace automates traffic manipulation.

## 🧭 Task 9: Connecting Through the Proxy (FoxyProxy)

To intercept browser traffic, configure Firefox to use Burp.

### 🦊 FoxyProxy Configuration

### Add New Proxy

```text
Title: Burp
IP: 127.0.0.1
Port: 8080
```

![](https://github.com/Deeptig9138/CyberSecurity101---THM/blob/main/screenshots/M8/m8.22.png)

### Activate Proxy

Select:

```text
Burp
```

from the FoxyProxy menu.

### Enable Interception

In Burp:

```text
Proxy → Intercept
```

Turn interception on.

### Test

Visit:

```text
http://MACHINE_IP/
```

The request should appear inside Burp.

### Important Notes

When intercept is enabled:

```text
Browser requests pause until forwarded.
```

Always remember to disable interception when finished.

## 🧭 Task 11: The Burp Suite Browser

Burp includes a built-in Chromium browser.

Launch it from:

```text
Open Browser
```

inside the Proxy tab.

### Benefits

* Automatically configured
* No FoxyProxy required
* Immediate interception support

### Linux Sandbox Issue

Running Burp as root may cause startup problems.

Solutions:

#### Recommended

Run Burp as a non-root user.

#### Quick Fix

Enable:

```text
Settings → Tools → Burp Browser
→ Allow browser to run without sandbox
```

Use cautiously.

## 🧭 Task 13: Proxying HTTPS

HTTPS traffic requires trusted certificates.

### Problem

Without Burp's certificate:

```text
Certificate Warning
```

appears in the browser.

### Solution

Visit:

```text
http://burp/cert
```

Download:

```text
cacert.der
```

### Import Into Firefox

Navigate to:

```text
about:preferences
```

Search:

```text
Certificates
```

Then:

```text
View Certificates → Import
```

### Trust the Certificate

Enable:

```text
Trust this CA to identify websites
```

Burp can now intercept HTTPS traffic.

## 🧭 Task 14: Example Attack

Let's examine a simplified XSS attack.

### 🎯 Target

Support Form:

```text
/ticket/
```

### Client-Side Validation

The email field only accepts valid email characters.

Example payload:

```html
<script>alert("Succ3ssful XSS")</script>
```

Normally blocked.

### Bypass Using Burp

#### Step 1

Submit valid data:

```text
pentester@example.thm
```

#### Step 2

Intercept request.

#### Step 3

Replace email value with:

```html
<script>alert("Succ3ssful XSS")</script>
```

#### Step 4

URL Encode payload:

```text
Ctrl + U
```

#### Step 5

Forward request.

### Result

The application reflects the payload and executes:

```javascript
alert("Succ3ssful XSS")
```

This demonstrates:

```text
Reflected XSS
```

### 🛡 Security Lesson

Client-side validation alone is never sufficient.

Attackers can easily bypass browser restrictions using Burp Suite.

Always validate input server-side.
