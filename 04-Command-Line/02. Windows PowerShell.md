## 🧭 Task 2: What Is PowerShell

From the official Microsoft page:

> *“PowerShell is a cross-platform task automation solution made up of a command-line shell, a scripting language, and a configuration management framework.”*

PowerShell is a powerful tool from Microsoft designed for task automation and configuration management. It combines a command-line interface and a scripting language built on the .NET framework.

### 🏛️ A Brief History of PowerShell

In the early 2000s, traditional tools like `cmd.exe` and batch files fell short in automating and managing complex enterprise Windows environments. Microsoft needed a tool that could handle sophisticated administrative tasks and interact with modern Windows APIs.

Jeffrey Snover, a Microsoft engineer, realized that Windows and Unix handled system operations differently:

* **Unix** treated everything as text files.
* **Windows** used structured data and APIs.

This difference made porting Unix tools to Windows impractical. Snover's solution was to develop an **object-oriented approach**, combining scripting simplicity with the power of the .NET framework. Released in 2006, PowerShell allowed administrators to automate tasks more effectively by manipulating objects. In 2016, Microsoft released **PowerShell Core**, an open-source and cross-platform version that runs on Windows, macOS, and Linux.

### 🧠 The Power in PowerShell: Object-Oriented Design

Unlike older text-based command-line tools that process and output data as plain text streams, PowerShell is **object-oriented**.

In programming, an **object** represents an item with:

* **Properties:** Data or characteristics (e.g., a car's `Color`, `Model`, or `FuelLevel`).
* **Methods:** Actions or functions (e.g., `Drive()`, `HonkHorn()`, or `Refuel()`).

In PowerShell, objects encapsulate data and functionality. An object can contain file names, usernames, or sizes as **properties**, and carry functions (**methods**) such as copying a file or stopping a process. When a command—known as a **cmdlet** (pronounced *command-let*)—is run in PowerShell, it returns objects that retain their properties and methods. This allows for powerful data manipulation without requiring additional text parsing.

## 🧭 Task 3: PowerShell Basics

### 🛠️ Basic Syntax: Verb-Noun

PowerShell commands are called `cmdlets`. They follow a strict, consistent **Verb-Noun** naming convention that describes their behavior:

* **Verb:** Describes the action being performed (e.g., `Get` to retrieve data, `Set` to configure details).
* **Noun:** Specifies the target object on which the action is performed (e.g., `Content`, `Location`).

Examples include:

* `Get-Content`: Retrieves the content of a file and displays it in the console.
* `Set-Location`: Changes the current working directory.

### 🔍 Discovery and Help Cmdlets

* `Get-Command`: Lists all available cmdlets, functions, aliases, and scripts in the current session. You can filter the results by property values using parameters like `-CommandType "Function"`.
* `Get-Help`: Provides detailed information about cmdlets, including usage, syntax maps, parameters, and examples (e.g., appending `-examples` to your help command).
* `Get-Alias`: Lists shortcuts or alternative names for cmdlets. This allows users familiar with traditional command-line tools to use familiar syntax (e.g., `dir` is an alias for `Get-ChildItem`, and `cd` or `chdir` are aliases for `Set-Location`).

### 📥 Finding and Downloading Cmdlets

You can extend PowerShell's functionality by downloading additional collections of cmdlets called **modules** from online repositories like the PowerShell Gallery.

* `Find-Module`: Searches online repositories for modules. You can use wildcards (`*`) to search for partial matches using the syntax: `Cmdlet -Property "pattern*"`.
* `Install-Module`: Downloads and installs the identified module from the repository onto the system.

*(Note: These repository features require an internet connection and will not run in an isolated lab environment without access).*

## 🧭 Task 4: Navigating the File System and Working with Files

PowerShell provides unified cmdlets to handle the creation and management of both files and directories, simplifying traditional architectures that relied on completely separate commands.

### 📂 File Management Core Commands

* `Get-ChildItem`: Lists the files and directories in a specified location via the `-Path` parameter. If no path is provided, it displays the contents of the current working directory.
* `Set-Location`: Navigate to a different path destination (similar to the legacy `cd` command).
* `New-Item`: Creates a new item on the system. You must explicitly specify the directory path and the item structure using the `-ItemType` parameter (e.g., `"Directory"` or `"File"`).
* `Remove-Item`: Deletes both files and directories seamlessly (replacing separate legacy tools like `rmdir` and `del`).
* `Copy-Item`: Duplicates files and folders to a specified destination.
* `Move-Item`: Moves or renames resources across filesystem paths.
* `Get-Content`: Reads and renders the literal contents of a file inside the terminal window.

## 🧭 Task 5: Piping, Filtering, and Sorting Data

**Piping** (represented by the `|` symbol) takes the output of one command and sends it as input to another. In PowerShell, this process passes entire **objects** with their data, properties, and methods intact.

### 📊 Filtering and Sorting Operations

* `Sort-Object`: Orders objects based on specified properties (e.g., `Get-ChildItem | Sort-Object Length` sorts items numerically by their file size). You can add the `-Descending` flag to reverse the sorting order.
* `Where-Object`: Evaluates pipeline data against explicit conditions, passing through only the items that meet the criteria.
* `Select-Object`: Isolates specific object properties for display (e.g., `Select-Object Name,Length`) or limits the quantity of returned rows (e.g., `-First 1`).
* `Select-String`: Searches for literal text patterns or complex regular expressions (`regex`) inside files, operating similarly to `grep` in Unix or `findstr` in legacy CMD.

### ⚙️ Pipeline Comparison Operators

PowerShell utilizes specialized script-safe comparison operators to evaluate data:

* `-eq` : Equal to
* `-ne` : Not equal to
* `-gt` : Greater than (strict comparison)
* `-ge` : Greater than or equal to
* `-lt` : Less than (strict comparison)
* `-le` : Less than or equal to
* `-like` : Wildcard pattern matching (e.g., checking names matching `"ship*"`)

### ✅ Answer

#### How would you retrieve the items in the current directory with size greater than 100? [for the sake of this question, avoid the use of quotes (" or ') in your answer]

```text
Get-ChildItem | Where-Object -Property Length -gt 100
```

## 🧭 Task 6: System and Network Information

PowerShell offers dedicated cmdlets for system administrators and IT professionals to audit core machine settings.

### 🖥️ Asset and Profile Discovery

* `Get-ComputerInfo`: Gathers a comprehensive system configuration profile, including detailed operating system versions, hardware profiles, and BIOS attributes (retrieving far more details than legacy tools like `systeminfo`).
* `Get-LocalUser`: Lists local computer accounts, showing their current account status (`Enabled` state) and systemic descriptions.

### 🌐 Network Configuration Mapping

* `Get-NetIPConfiguration`: Lists configuration status profiles for active network interfaces, including active IP addresses, current DNS servers, and network gateways.
* `Get-NetIPAddress`: Displays comprehensive binding telemetry details for every configured IP address on the machine, including loopback paths and inactive configurations.

## 🧭 Task 7: Real-Time System Analysis

For incident response and threat hunting, PowerShell includes specialized cmdlets to audit dynamic system behavior and identify active anomalies.

### 🔍 Dynamic Forensic Discovery Toolkit

* `Get-Process`: Captures structural runtime details for all currently executing processes, tracking CPU/memory consumption metrics, process IDs (`Id`), and execution handles.
* `Get-Service`: Reviews system services status profiles (`Running`, `Stopped`, or `Paused`), helping analysts look for unauthorized background tasks.
* `Get-NetTCPConnection`: Displays active TCP network connection layers, capturing local/remote sockets, network states, and the process that initialized it via the `OwningProcess` identifier.
* `Get-FileHash`: Computes secure cryptographic checksum signatures (such as `SHA256`) to confirm file integrity and detect potential tampering.
* `Get-Item -Stream *`: Audits hidden Alternate Data Streams (`ADS`) attached to NTFS files. This helps identify hidden files or payloads appended behind the primary file stream (the default data stream is marked as `:$DATA`).

## 🧭 Task 8: Scripting

**Scripting** is the process of writing and executing a series of commands saved in a text file (a script) to automate repetitive or complex tasks.

### 🛡️ Cybersecurity Scripting Roles

* **Blue Team (Defensive Security):** Incident responders and threat hunters use scripts to automate large-scale log analysis, detect system anomalies, scan for intrusion indicators, and inspect systems for indicators of compromise (IOCs).
* **Red Team (Offensive Security):** Penetration testers and ethical hackers automate system enumeration, execute remote commands, and craft obfuscated execution chains to test defensive resilience.
* **System Administrators:** Automate configuration integrity checks, enforce security baseline compliance policies, and monitor machine health across large enterprise environments.

### 📡 Remote Task Orchestration via `Invoke-Command`

The `Invoke-Command` cmdlet allows administrators and security professionals to run scripts or command sequences across remote target computers simultaneously:

* **Running a Script File:** Using the `-FilePath` parameter to execute a local script file on a remote host (e.g., `-ComputerName Server01`).
* **Running Inline Commands:** Using the `-ScriptBlock { ... }` parameter to execute a block of commands remotely. The execution returns the results to your local session as if you were running them directly on the remote machine.

### ✅ Answer

#### What is the syntax to execute the command Get-Service on a remote computer named "RoyalFortune"? Assume you don't need to provide credentials to establish the connection. [for the sake of this question, avoid the use of quotes (" or ') in your answer]

```text
Invoke-Command -ComputerName RoyalFortune -ScriptBlock {Get-Service}
```

## 📝 Key Takeaways

* **Object-Oriented Architecture:** PowerShell processes data as structured .NET objects with distinct properties and methods, rather than relying on flat plain text strings.
* **Predictable Syntax:** Native cmdlets use a clear `Verb-Noun` naming structure, making it intuitive to find and use new commands.
* **Built-in Discovery Tools:** Discovery features like `Get-Command`, `Get-Help`, and `Get-Alias` provide comprehensive documentation and syntax examples directly inside the shell.
* **Unified Object Pipeline:** Using the pipe operator (`|`) allows you to accurately sort, filter, and extract data using specialized tools like `Where-Object` and `Select-Object`.
* **Dynamic Auditing:** Built-in cmdlets provide immediate visibility into active processes (`Get-Process`), network connections (`Get-NetTCPConnection`), and file integrity signatures (`Get-FileHash`).
