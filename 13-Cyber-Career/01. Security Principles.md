# 🛡️ Security Principles

Security is more than simply installing antivirus software or setting strong passwords. Effective security requires understanding what you're protecting, who you're protecting it from, and which controls best reduce risk.

One important reality is:

> Perfect security does not exist.

Instead, organizations aim to improve their security posture and make attacks more difficult, expensive, and time-consuming for adversaries.

## 🧭 Task 2: CIA Triad

The foundation of information security is the:

> CIA Triad

CIA stands for:

* Confidentiality
* Integrity
* Availability

### 🔒 Confidentiality

Confidentiality ensures that only authorized individuals can access sensitive information.

Examples:

* Medical records
* Passwords
* Credit card details
* Financial reports

#### Example

When shopping online, your credit card information should only be accessible to the payment processor.

If unauthorized parties can access it:

```text
Confidentiality is lost
```

### ✅ Integrity

Integrity ensures that data remains accurate and unmodified.

Any unauthorized changes should be:

* Prevented
* Detected

#### Example

If an attacker changes your delivery address after checkout:

```text
Integrity has been compromised
```

### 🌐 Availability

Availability ensures that systems remain accessible when needed.

Examples:

* Websites
* Databases
* Cloud services
* Medical systems

#### Example

If an online store goes offline:

```text
Customers cannot place orders
```

Availability has been impacted.

### 🏥 CIA in Healthcare

#### Confidentiality

Patient records must remain private.

Unauthorized disclosure may violate laws and regulations.

#### Integrity

Medical records must remain accurate.

Incorrect records could result in:

* Wrong medication
* Incorrect diagnosis
* Life-threatening situations

#### Availability

Doctors require immediate access to records.

Unavailable systems can delay treatment and increase risk.

### 🔐 Beyond CIA

Two additional security properties are often required.

#### Authenticity

Authenticity ensures data genuinely originates from the claimed source.

Example:

```text
Did the customer actually place this order?
```

#### Nonrepudiation

Nonrepudiation ensures the sender cannot later deny their actions.

Example:

```text
A customer cannot deny submitting an order.
```

This is critical in:

* Banking
* E-commerce
* Healthcare
* Legal systems

### 🧩 Parkerian Hexad

In 1998, Donn Parker expanded the CIA Triad into six security elements.

#### The Six Elements

1. Availability
2. Utility
3. Integrity
4. Authenticity
5. Confidentiality
6. Possession

### Utility

Utility measures whether information is actually usable.

Example:

An encrypted drive without the decryption key.

```text
Data exists but cannot be used.
```

### Possession

Possession focuses on ownership and control.

Example:

An attacker steals a backup drive.

```text
Possession is lost
```

even if confidentiality remains intact.

## 🧭 Task 3: DAD Triad

The opposite of CIA is:

> DAD

DAD stands for:

* Disclosure
* Alteration
* Destruction / Denial

### 📤 Disclosure

Opposite of:

```text
Confidentiality
```

Example:

Patient records leaked online.

### ✏️ Alteration

Opposite of:

```text
Integrity
```

Example:

An attacker modifies medical records.

### 🚫 Destruction / Denial

Opposite of:

```text
Availability
```

Example:

Attackers disable critical systems and services.

### ⚖️ Security Balance

Strong security requires balancing all three areas.

Too much emphasis on one area can negatively affect the others.

Example:

Excessive security controls may reduce availability.

### ✅ Answers

**Customer records were stolen and published online. What attack is this?**

```text
Disclosure
```

**Attackers disabled the primary and backup power systems, shutting down the network. What attack is this?**

```text
Destruction/Denial
```

## 🧭 Task 4: Fundamental Security Models

Security models provide structured approaches for enforcing security requirements.

This room introduces:

* Bell-LaPadula
* Biba
* Clark-Wilson

### 🔒 Bell-LaPadula Model

Focus:

```text
Confidentiality
```

#### Rule 1: No Read Up

Users cannot read information above their clearance level.

```text
No Read Up
```

#### Rule 2: No Write Down

Users cannot write sensitive information to lower security levels.

```text
No Write Down
```

#### Memory Aid

```text
Read Down, Write Up
```

### 🛡️ Biba Model

Focus:

```text
Integrity
```

#### Rule 1: No Read Down

High-integrity users should not read lower-integrity data.

```text
No Read Down
```

#### Rule 2: No Write Up

Lower-integrity users cannot modify higher-integrity data.

```text
No Write Up
```

#### Memory Aid

```text
Read Up, Write Down
```

### ⚙️ Clark-Wilson Model

Focus:

```text
Integrity
```

through controlled processes.

#### Key Components

| Component | Description |
|------------|-------------|
| CDI | Constrained Data Item |
| UDI | Unconstrained Data Item |
| TP | Transformation Procedure |
| IVP | Integrity Verification Procedure |

#### Goal

Ensure all modifications occur through approved procedures.

## 🧭 Task 5: Defence-in-Depth

Defence-in-Depth means:

> Multiple layers of security protecting the same asset.

### 🏠 Example

Protecting valuables in a home:

1. Locked drawer
2. Locked room
3. Locked apartment door
4. Locked building gate
5. Security cameras

An attacker must bypass multiple controls.

### Benefits

* Slows attackers
* Increases detection opportunities
* Reduces single points of failure
* Improves resilience

## 🧭 Task 6: ISO/IEC 19249

ISO/IEC 19249 defines architectural and design principles for secure systems.

### 🏗️ Architectural Principles

#### 1️⃣ Domain Separation

Separate resources into security domains.

Example:

```text
CPU Privilege Rings
```

#### 2️⃣ Layering

Implement security across multiple layers.

Example:

```text
OSI Model
```

#### 3️⃣ Encapsulation

Hide internal implementation details.

Example:

```text
APIs
```

instead of direct database access.

#### 4️⃣ Redundancy

Duplicate critical components.

Examples:

* RAID
* Backup power supplies
* Failover systems

#### 5️⃣ Virtualization

Run multiple isolated environments on shared hardware.

Benefits:

* Isolation
* Sandboxing
* Improved security boundaries

### 🎯 Design Principles

#### 1️⃣ Least Privilege

Provide only the permissions required.

Example:

```text
Read access without write access
```

#### 2️⃣ Attack Surface Minimization

Reduce unnecessary exposure.

Example:

```text
Disable unused services
```

#### 3️⃣ Centralized Parameter Validation

Validate inputs consistently from a central location.

Protects against:

* Injection attacks
* RCE
* DoS

#### 4️⃣ Centralized Security Services

Centralize security functions.

Examples:

* Authentication servers
* Identity management systems

#### 5️⃣ Error and Exception Handling

Systems should fail securely.

Example:

A firewall should:

```text
Block traffic if it crashes
```

instead of allowing all traffic.

### ✅ Answers

**Turning off an insecure, unnecessary server demonstrates which principle?**

```text
2
```

**Granting a sales employee access only to product information demonstrates which principle?**

```text
1
```

**Extensive code handling outages and failures demonstrates which principle?**

```text
5
```

## 🧭 Task 7: Zero Trust vs Trust but Verify

Modern security heavily relies on trust models.

### 🤝 Trust but Verify

Principle:

```text
Trust entities, but continuously verify actions.
```

Examples:

* Logging
* Monitoring
* IDS
* IPS
* Proxy auditing

### 🚫 Zero Trust

Principle:

```text
Never Trust, Always Verify
```

### Core Concepts

* Every user is untrusted by default.
* Every device is untrusted by default.
* Authentication is always required.
* Authorization is always required.

### Benefits

* Limits insider threats
* Reduces breach impact
* Improves visibility

### 🔬 Microsegmentation

A common Zero Trust technique.

Creates very small network segments.

Example:

```text
One host per segment
```

Access requires verification between segments.

## 🧭 Task 8: Vulnerability vs Threat vs Risk

These three terms are often confused.

### ⚠️ Vulnerability

A weakness.

Example:

```text
Unpatched software
```

### 👤 Threat

A potential danger capable of exploiting a vulnerability.

Example:

```text
Attackers targeting the vulnerable software
```

### 📊 Risk

The likelihood and impact of successful exploitation.

Risk considers:

* Probability
* Business impact

#### Example

### Vulnerability

```text
Glass windows
```

### Threat

```text
Someone breaking the glass
```

### Risk

```text
Likelihood and consequences of the break-in
```

### Healthcare Example

#### Vulnerability

Database software flaw.

#### Threat

Public exploit code released.

#### Risk

Potential compromise of patient records.
