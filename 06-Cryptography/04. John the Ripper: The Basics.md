## Task 2: Basic Terms

### What Are Hashes?

A **hash** is a fixed-length representation of data generated by a hashing algorithm.

### Example

| Input     | MD5 Hash                         |
| --------- | -------------------------------- |
| polo      | b53759f3ce692de7aff1b5779d3964da |
| polomints | 584b6e4f4586e136bc280f27f9c64f3b |

Even though the inputs have different lengths, both hashes are exactly 32 hexadecimal characters.

### Why Are Hashes Secure?

Hash functions are designed as **one-way functions**:

#### Easy

Input → Hash

#### Difficult

Hash → Original Input

This is why passwords are stored as hashes rather than plaintext.

### P vs NP (Simplified)

#### P (Polynomial Time)

Problems that can be solved efficiently.

**Example:** Sorting a list.

#### NP (Non-deterministic Polynomial Time)

Problems where verifying a solution is easy, but finding the solution is difficult.

Hashing belongs to the easy category, while reversing a hash falls into the difficult category.

### Where John the Ripper Comes In

Although hashes cannot realistically be reversed, they can be **cracked**.

#### Dictionary Attack Process

1. Take a word from a wordlist.
2. Hash it.
3. Compare it with the target hash.
4. If they match, the password is found.

John automates this process.

### Answer

**Q:** What is the most popular extended version of John the Ripper?

**A:** `Jumbo John`

```bash
john --format=raw-md5 \
```

Search specific formats:

## Task 6: Cracking /etc/shadow Hashes

### Linux Password Storage

Passwords are stored in:

```bash
/etc/shadow
```

User account information is stored in:

```bash
/etc/passwd
```

### Unshadow

John requires both files combined.

#### Syntax

```bash
unshadow passwd shadow > unshadowed.txt
```

#### Example

```bash
unshadow local_passwd local_shadow > unshadowed.txt
```

### Cracking

```bash
john \
--wordlist=/usr/share/wordlists/rockyou.txt \
--format=sha512crypt \
unshadowed.txt
```

### Answer

**Q:** What is the root password?

**A:** `1234`

## Task 7: Single Crack Mode

### What Is Single Crack Mode?

Instead of using a wordlist, John generates passwords from known user information.

### Word Mangling

Given username:

```text
Markus
```

Possible guesses:

```text
Markus1
Markus2
Markus!
MARKus
```

### GECOS Information

John can also use data from:

* Full name
* Home directory
* Phone numbers
* Office information

stored inside `/etc/passwd`.

### Syntax

```bash
john --single --format=[format] hashfile
```

Example:

```bash
john --single --format=raw-sha256 hashes.txt
```

### File Format Requirement

Before:

```text
1efee03cdcb96d90ad48ccc7b8666033
```

After:

```text
mike:1efee03cdcb96d90ad48ccc7b8666033
```

## Task 8: Custom Rules

### What Are Custom Rules?

Custom rules allow you to generate password variations automatically.

Useful when password policies enforce:

* Uppercase letters
* Numbers
* Symbols

### Example Pattern

Target password:

```text
Polopassword1!
```

### Rule Components

#### Capitalize

```text
c
```

#### Append

```text
Az
```

#### Prepend

```text
A0
```

### Character Sets

| Character Set | Meaning           |
| ------------- | ----------------- |
| [0-9]         | Numbers           |
| [A-Z]         | Uppercase letters |
| [a-z]         | Lowercase letters |
| [!£$%@]       | Symbols           |

### Example Rule

```text
[List.Rules:PoloPassword]
cAz"[0-9][!£$%@]"
```

Meaning:

* Capitalize first letter
* Append a number
* Append a symbol

### Using Custom Rules

```bash
john \
--wordlist=rockyou.txt \
--rule=PoloPassword \
hash.txt
```

### Answers

#### What do custom rules allow us to exploit?

`Password complexity predictability`

#### What rule would we use to add all capital letters to the end of the word?

```text
Az"[A-Z]"
```

#### What flag would we use to call a custom rule called THMRules?

```bash
--rule=THMRules
```

## Task 11: Cracking SSH Keys with John

### SSH Private Keys

SSH key authentication commonly uses:

```text
id_rsa
```

Private keys may be protected with a passphrase.

### ssh2john

Convert the SSH key into a crackable format.

#### AttackBox

```bash
python3 /opt/john/ssh2john.py id_rsa > id_rsa_hash.txt
```

#### Kali

```bash
python /usr/share/john/ssh2john.py id_rsa > id_rsa_hash.txt
```

### Cracking

```bash
john \
--wordlist=/usr/share/wordlists/rockyou.txt \
id_rsa_hash.txt
```
