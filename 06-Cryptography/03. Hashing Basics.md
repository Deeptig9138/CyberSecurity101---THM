# #️⃣ Hashing Basics

This room introduces the fundamentals of **Hashing**, one of the most important concepts in cryptography and cybersecurity.

Unlike encryption, hashing is designed to be a **one-way process**. It is used to verify data integrity, securely store passwords, validate downloads, and authenticate data.

Hashing is used extensively across modern systems, including:

- Password Authentication
- File Integrity Verification
- Digital Signatures
- HMAC Authentication
- Malware Analysis
- Password Cracking

## 🧭 Task 2: Hash Functions

### 🔑 What Is a Hash Function?

A hash function takes input data of any size and produces a fixed-size output called a:

> **Hash Value** (or Digest)

### Hashing Process

```text
Input Data
     ↓
Hash Function
     ↓
Hash Value
```

Unlike encryption:

❌ No decryption key exists

❌ Original data cannot practically be recovered

✅ Same input always produces the same output

✅ Tiny changes create completely different hashes

### 📌 Example

Two files differ by only one bit:

```text
T = 01010100
U = 01010101
```

Yet their hashes are completely different.

### MD5

```text
T → b9ece18c950afbfa6b0fdbfa4ff731d3

U → 4c614360da93c0a041b22e537de151eb
```

### SHA1

```text
T → c2c53d66948214258a26ca9ca845d7ac0c17f8e7

U → b2c7c0caa10a0cca5ea7d69e54018ae0c0389dd6
```

### SHA256

```text
T → e632b7095b0bf32c260fa4c539e9fd7b...
U → a25513c7e0f6eaa80a3337ee18081b9e...
```

A one-bit difference completely changes the output.

This property is known as:

> Avalanche Effect

### 🌍 Why Hashing Matters

Hashing is used constantly behind the scenes.

Examples include:

#### Password Verification

When you log into a website:

```text
Password
   ↓
Hash
   ↓
Compare with Stored Hash
```

The actual password should never be stored.

#### File Verification

Used to confirm:

- Download integrity
- No corruption
- No tampering

### ⚠️ Hash Collisions

A collision occurs when:

```text
Input A ≠ Input B
```

but

```text
Hash(A) = Hash(B)
```

#### Why Collisions Must Exist

Hash outputs are fixed size.

Possible inputs are effectively unlimited.

This is explained by:

> Pigeonhole Principle

Example:

```text
4-bit hash
```

Produces only:

```text
2⁴ = 16
```

possible outputs.

Eventually different inputs must share outputs.

### 🚨 Insecure Hash Functions

#### MD5

Broken due to collision attacks.

#### SHA1

Also broken due to practical collision attacks.

Examples:

- MD5 Collision Demo
- SHAttered Attack

Neither should be trusted for:

❌ Password storage

❌ Integrity protection

#### What is the SHA256 hash of `passport.jpg`?

```text
77148c6f605a8df855f2b764bcc3be749d7db814f5f79134d2aa539a64b61f02
```


## 🧭 Task 3: Insecure Password Storage

Hashing is commonly used for:

> Authentication

When a user logs in, the system verifies knowledge of a password without storing the actual password.

### 🚨 Common Insecure Practices

#### 1️⃣ Storing Plaintext Passwords

Worst possible option.

Example:

### RockYou Breach

Passwords were stored in plaintext.

Result:

```text
14,344,392
```

passwords leaked.

Common passwords included:

```text
123456
password
iloveyou
abc123
```

#### 2️⃣ Using Weak Encryption

Example:

### Adobe Breach

Problems:

- Deprecated encryption
- Password hints stored openly

Attackers recovered many passwords.

#### 3️⃣ Using Weak Hashing

Example:

### LinkedIn Breach (2012)

Problems:

- SHA1 used
- No salt used

Attackers cracked millions of passwords.

### 🧂 What Is Salting?

A salt is:

> Random data added to a password before hashing

Example:

```text
Password:
AL4RMc10k

Salt:
Y4UV*^(=go_!

Combined:
AL4RMc10kY4UV*^(=go_!
```

Then hashed.

This makes identical passwords produce different hashes.

## 🧭 Task 4: Secure Password Storage

### 🔒 Why Hash Passwords?

Instead of storing:

```text
Password
```

Store:

```text
Hash(Password)
```

If the database leaks:

❌ Attackers do not immediately know passwords.

### 🌈 Rainbow Tables

A Rainbow Table is:

> A precomputed list of hashes and matching passwords.

Example:

| Hash | Password |
|--------|----------|
| e10adc3949ba59abbe56e057f20f883e | 123456 |
| e99a18c428cb38d5f260853678922e03 | abc123 |
| d0199f51d2728db6011945145a1b607a | basketball |

### 🧂 Protecting Against Rainbow Tables

Use:

#### Unique Salt Per User

Even if two users choose:

```text
Password123
```

their hashes become different.

### Recommended Password Hashing Algorithms

✅ Argon2

✅ Scrypt

✅ Bcrypt

✅ PBKDF2

Avoid:

❌ MD5

❌ SHA1

### Why Not Encrypt Passwords?

Encryption requires:

```text
Encryption Key
```

If attackers steal the key:

```text
All passwords become recoverable
```

Hashing avoids this problem.

## 🧭 Task 5: Recognising Password Hashes

### 🔎 Hash Identification

Recognizing hash types helps determine:

- Storage method
- Cracking approach
- Hashcat mode

Tools:

- hashID
- Hashcat
- John the Ripper

Always combine:

✅ Tools

✅ Context

### 🐧 Linux Password Hashes

Stored in:

```text
/etc/shadow
```

Format:

```text
$prefix$options$salt$hash
```

### Common Linux Hash Prefixes

| Prefix | Algorithm |
|----------|----------|
| `$y$` | yescrypt |
| `$gy$` | gost-yescrypt |
| `$7$` | scrypt |
| `$2a$`, `$2b$`, `$2y$` | bcrypt |
| `$6$` | sha512crypt |
| `$1$` | md5crypt |

### Example

```text
$y$j9T$76UzfgEM5PnymhQ7TlJey1$/OOS...
```

Breakdown:

```text
y     → yescrypt
j9T   → parameter
76... → salt
/O... → hash
```

### 🪟 Windows Password Hashes

Windows commonly uses:

> NTLM

Stored in:

```text

(Security Accounts Manager)
```

Tools like:

```text

```

## 🧭 Task 6: Password Cracking

Hashes cannot be decrypted.

Instead:

```text
Guess Password
      ↓
Hash Guess
      ↓
Compare
```

Repeat until matched.

### 🔨 Common Cracking Tools

#### Hashcat

GPU-accelerated.

#### John the Ripper

CPU-focused.

### Basic Hashcat Syntax

```bash
hashcat -m <type> -a <attack> hashfile wordlist
```

Example:

```bash
hashcat -m 1400 -a 0 hash.txt rockyou.txt
```

### GPU Advantages

Modern GPUs contain:

```text
Thousands of cores
```

allowing rapid hash calculations.

Some algorithms resist GPU acceleration:

- Bcrypt
- Scrypt
- Argon2

## 🧭 Task 7: Hashing for Integrity Checking

### 📦 File Integrity Verification

Hashes help verify:

- Downloads
- Backups
- System files
- Software packages

Example:

```bash
sha256sum file.iso
```

Compare output with the vendor-provided hash.

Matching hashes indicate:

✅ Same file

✅ No corruption

✅ No tampering

### 📑 Duplicate File Detection

If:

```text
Hash(File A) = Hash(File B)
```

then the files are almost certainly identical.

Useful for:

- Storage cleanup
- Malware investigations
- Backup verification

### 🔐 HMAC

HMAC stands for:

> Hash-based Message Authentication Code

Provides:

✅ Integrity

✅ Authenticity

#### Components

```text
Message
+
Secret Key
+
Hash Function
```

Example:

```text
HMAC(K,M)
```

Only someone with the secret key can generate the correct HMAC.

### Common Uses

- API Authentication
- TLS
- JWT Signing
- Secure Communications

### 🔄 Hashing vs Encoding vs Encryption

#### #️⃣ Hashing

```text
One-way
```

Purpose:

- Password storage
- Integrity checking

Examples:

```text
MD5
SHA1
SHA256
SHA512
```

#### 🔤 Encoding

```text
Reversible
```

Purpose:

- Data representation

Examples:

```text
ASCII
UTF-8
Base64
```

Example:

```bash
echo "TryHackMe" | base64

VHJ5SGFja01lCg==
```

Decode:

```bash
base64 -d
```

#### 🔐 Encryption

```text
Reversible
```

Requires:

```text
Key
```

Purpose:

- Confidentiality

Examples:

```text
AES
RSA
ECC
```

### ✅ Final Answer

#### Decode:

```text
RU5jb2RlREVjb2RlCg==
```

```bash
base64 -d decode-this.txt
```

## 📝 Key Takeaways

- Hash functions generate fixed-length outputs.
- Hashing is one-way and cannot be reversed.
- Small input changes create drastically different hashes.
- MD5 and SHA1 are no longer considered secure.
- Passwords should be hashed, salted, and stored using strong algorithms.
- Rainbow tables attack unsalted hashes.
- Linux stores password hashes in `/etc/shadow`.
- Hashcat and John the Ripper are popular cracking tools.
- Hashes verify file integrity.
- HMAC provides authenticity and integrity.
