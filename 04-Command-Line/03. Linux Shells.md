# 🐚 Linux Shells

Most modern computer users rely heavily on a Graphical User Interface (GUI) to complete tasks using visual menus and mouse clicks. However, nearly every operational task can be executed more efficiently and with fewer system resources by typing commands directly into the Command-Line Interface (CLI).

The tool that makes this possible is the **shell**. The shell acts as the direct facilitator or middleman between the user and the operating system's kernel.

## 🎯 Learning Objectives

By completing this room, you will learn:

* How to interact with a Linux shell via the Command-Line Interface (CLI).
* How to run essential, everyday Linux shell commands.
* The distinct characteristics and differences between popular Linux shells (Bash, Fish, and Zsh).
* The foundational components of shell scripting (variables, loops, conditional statements, and comments) to automate tasks.

## 🧭 Task 2: How To Interact With a Shell?

When opening a terminal connection to a Linux virtual machine, you are greeted by a visual text prompt indicating that the system is ready to accept commands.

```bash
user@tryhackme:~$
```

While most modern Linux distributions deploy **Bash** as their native, out-of-the-box shell environment, your target directory and precise configuration context dictate how you navigate.

### 🧰 Core Interaction Commands

* `pwd` (**Print Working Directory**): Displays the exact, absolute path of the directory you are currently working within. By default, opening a terminal drops you directly into your account's dedicated home space (e.g., `/home/user`).
* `cd` (**Change Directory**): Updates your active terminal location to a new target path (e.g., `cd Desktop` drops your context into the local Desktop folder).
* `ls` (**List Directory Contents**): Acts as your digital eyes in the terminal. It enumerates all visible files and folders sitting inside your current working directory.
* `cat` (**Concatenate**): Reads a target file and streams its literal text content straight into the console window for review.
* `grep` (**Global Regular Expression Print**): A highly powerful search tool used to parse massive chunks of data or large files. By specifying a distinct keyword or pattern, `grep` isolates and outputs only the explicit lines containing that phrase.

## 🧭 Task 3: Types of Linux Shells

Linux does not restrict you to a single shell interface. Multiple shells can be installed concurrently on a single machine.

To determine the explicit shell currently handling your active terminal session, check the system environment variable:

```bash
user@tryhackme:~$ echo $SHELL
/bin/bash
```

To see every valid login shell installed on your current operating system, read the contents of the `/etc/shells` configuration file:

```bash
user@tryhackme:~$ cat /etc/shells
/bin/sh
/bin/bash
/bin/dash
/bin/zsh
```

You can switch your active interface instantly by typing the binary path of the desired shell (e.g., typing `zsh`). To permanently change your default login shell across new system restarts, utilize the `chsh` (change shell) utility with the `-s` parameter:

```bash
chsh -s /usr/bin/zsh
```

### 🐚 High-Profile Shell Breakdown

* **Bourne Again Shell (Bash):** The definitive legacy standard for modern Linux systems. It acts as an enhanced, backward-compatible replacement for the original Unix Bourne shell (`sh`), integrating features from other vintage environments (`ksh`, `csh`). It includes native tab-completion, history tracking (accessible via the `history` command or arrow keys), and highly stable, cross-compatible scripting profiles.
* **Friendly Interactive Shell (Fish):** A modern, user-centric shell designed strictly for interactive use. It introduces immediate syntax highlighting, auto-spell correction for commands, and smart suggestions based on history right out of the box. It avoids rigid legacy scripting standards in favor of absolute user clarity.
* **Z Shell (Zsh):** A highly sophisticated, feature-rich shell that combines the baseline stability of Bash with the modern interactive features of Fish. It supports heavy visual customization (via popular third-party frameworks like *Oh-My-Zsh*) and advanced tab-completion options driven by powerful extension plugins.

### 📊 Structural Attribute Comparison Matrix

| Feature Feature | Bash | Fish | Zsh |
| --- | --- | --- | --- |
| **Full Name** | Bourne Again Shell | Friendly Interactive Shell | Z Shell |
| **Scripting Capabilities** | High compatibility; extensively documented globally | Unique, non-standard minimalist syntax structures | Advanced scripting architecture built on traditional foundations |
| **Tab Completion** | Standard text completion options | Predictive inline suggestions based on execution history | High-level interactive tables powered by custom plugins |
| **Customization** | Low to basic configuration setups | Friendly web-based interactive tool panels | High-end ecosystem tailoring (*Oh-My-Zsh*) |
| **Syntax Highlighting** | Missing natively (requires independent tool logic) | Built-in out of the box | Available via plugin extension suites |
| **Auto-Correction** | Unsupported | Native out-of-the-box spelling correction | Native out-of-the-box spelling correction |

## 🧭 Task 4: Shell Scripting and Components

A **shell script** is a simple text file containing a sequence of commands. Instead of manually typing identical commands every time you perform a repetitive task, you can write them into a script file to execute them all at once.

### 🖋️ Initializing a Script File

1. Use a command-line text editor like `nano` to initialize your script file. Bash scripts traditionally use the `.sh` file extension:
```bash
user@tryhackme:~$ nano first_script.sh
```


2. **The Shebang (`#!`):** The very first line of any professional shell script must begin with a shebang. This is a special character sequence (`#!`) followed directly by the path of the interpreter the operating system should use to run the file. For a standard Bash script, write:
```bash
#!/bin/bash
```

### 🧱 Core Scripting Building Blocks

#### 1. Variables

Variables allow you to store data values (such as text strings, file paths, or system user inputs) inside a clean, reusable label. You can reference this label throughout your script rather than hardcoding complex data multiple times.

```bash
#!/bin/bash
echo "Hey, what’s your name?"
read name
echo "Welcome, $name"
```

*The `read` utility stops execution to accept user input, storing the typed text directly inside the `name` variable. Prefixing the variable with a `$` calls its stored value.*

#### 2. Executing the Script File

To run your new script, you must first configure the file system permissions to allow execution using the `chmod` command:

```bash
user@tryhackme:~$ chmod +x first_script.sh
```

To run the executable file within your active folder, append a `./` prefix directly in front of the filename:

```bash
user@tryhackme:~$ ./first_script.sh
```

> ⚠️ **Why use `./`?** The shell only looks for commands within directories specified in your system's global `PATH` environment variable. Because your active working directory is safely isolated from the global path, the `./` prefix tells the shell to look inside the *current directory* for the executable file.

#### 3. Loops

Loops automate repetitive actions by running a block of code multiple times. For example, a `for` loop can iterate through a sequence of numbers or a list of items:

```bash
#!/bin/bash
for i in {1..10}; do
    echo $i
done
```

*The iteration pointer `i` steps through the set from 1 to 10, running the `do` block on each pass until it reaches `done`.*

#### 4. Conditional Statements

Conditional statements use `if-then-else` logic to execute specific blocks of code only when explicit parameters match or evaluate to true.

```bash
#!/bin/bash
echo "Please enter your name first:"
read name

if [ "$name" = "Stewart" ]; then
    echo "Welcome Stewart! Here is the secret: THM_Script"
else
    echo "Sorry! You are not authorized to access the secret."
fi
```

*The conditional block must close with a backward `if` signature, written as `fi`.*

#### 5. Comments

Comments are clean notes written inside the script to explain complex logic. The shell completely ignores these lines during execution. Any line beginning with a `#` character (except for the primary shebang line) is treated as a comment:

```bash
# Asking the user to enter a value.
echo "Please enter your name first:"
```

## 🧭 Task 5: The Locker Script

Combining variables, loops, and conditional statements allows you to create secure, interactive authentication wrappers. The following script validates a user's bank details by checking their input against explicit access rules:

```bash
#!/bin/bash

# Defining the variables
username=""
companyname=""
pin=""

# Defining the loop to prompt for input fields sequentially
for i in {1..3}; do
        if [ "$i" -eq 1 ]; then
                echo "Enter your Username:"
                read username
        elif [ "$i" -eq 2 ]; then
                echo "Enter your Company name:"
                read companyname
        else
                echo "Enter your PIN:"
                read pin
        fi
done

# Checking if the user entered the correct details across all parameters
if [ "$username" = "John" ] && [ "$companyname" = "Tryhackme" ] && [ "$pin" = "7385" ]; then
        echo "Authentication Successful. You can now access your locker, John."
else
        echo "Authentication Denied!!"
fi
```

## 📝 Key Takeaways

* **The Shell Interface:** The shell acts as the primary text-based middleman translating user terminal commands directly into core kernel actions.
* **Unified Environment Choices:** Linux supports multiple shell installations (Bash, Fish, Zsh) inside the `/etc/shells` configuration file, allowing users to choose interfaces based on their customization and auto-correction needs.
* **Predictable Shell Flow:** Executable scripts depend on an explicit, leading shebang header (`#!/bin/bash`) to signal the correct system interpreter during startup.
* **Strict Script Execution Pathing:** Modern shells intentionally skip the local active directory when scanning for commands. You must explicitly prepend a `./` path to execute local scripts.
* **Logical Automation Components:** Scripts utilize variables to store temporary data, loops to handle repetitive actions, and conditional statements to enforce strict access logic.
