# 🗄️ SQL Fundamentals

SQL (Structured Query Language) is the standard language used to interact with relational databases. Databases are at the core of modern applications, storing everything from user accounts and authentication data to logs, transactions, and security events.

For cybersecurity professionals, understanding SQL is critical because databases are involved in:

* Web applications
* Authentication systems
* SIEM platforms
* Threat intelligence platforms
* Asset management systems
* Malware analysis tools
* Security monitoring solutions

From an offensive perspective, SQL knowledge helps identify and exploit vulnerabilities such as SQL Injection. From a defensive perspective, it enables analysts to investigate data, create reports, and secure database systems.

## 🧭 Task 2: Databases 101

Before learning SQL, it is important to understand how databases work.

### 📚 What is a Database?

A database is an organized collection of structured information that can be:

* Stored
* Retrieved
* Updated
* Analyzed

Examples include:

* User accounts and passwords
* Social media posts and comments
* Watch history on streaming platforms
* E-commerce transactions

### 🗃️ Types of Databases

There are two primary database categories:

#### Relational Databases (SQL)

Store data in structured tables.

Example:

| ID | First Name | Last Name | Email                                       |
| -- | ---------- | --------- | ------------------------------------------- |
| 1  | John       | Doe       | [john@example.com](mailto:john@example.com) |

Characteristics:

* Structured format
* Rows and columns
* Relationships between tables
* High data integrity

Common use cases:

* Banking
* E-commerce
* Inventory systems

#### NoSQL / Non-Relational Databases

Store data in flexible formats such as:

* Documents
* Key-value pairs
* Collections

Example:

```json
{
    "name": {
        "first": "Thomas",
        "last": "Anderson"
    },
    "occupation": ["The One"]
}
```

Characteristics:

* Flexible schema
* Handles varying data formats
* Highly scalable

Common use cases:

* Social media
* Large-scale content platforms
* Big data applications

### 📊 Tables, Rows, and Columns

Relational databases organize data using tables.

#### Table

A collection of related records.

Example:

```text
Books
```

#### Columns

Define what information each record contains.

Example:

```text
book_id
book_name
publication_date
```

Common data types:

| Type    | Description     |
| ------- | --------------- |
| INT     | Whole numbers   |
| VARCHAR | Text            |
| FLOAT   | Decimal numbers |
| DATE    | Dates           |

#### Rows

A row represents an individual record.

Example:

| book_id | book_name                  | publication_date |
| ------- | -------------------------- | ---------------- |
| 1       | Android Security Internals | 2014-10-14       |

### 🔑 Primary and Foreign Keys

#### Primary Key

Uniquely identifies each record.

Example:

```text
book_id
```

Rules:

* Must be unique
* Cannot be duplicated
* One primary key per table

#### Foreign Key

Creates relationships between tables.

Example:

```text
Books.author_id
```

references

```text
Authors.id
```

This allows a database to connect related records across tables.

### ✅ Answers

**What type of database should you consider using if the data you're going to be storing will vary greatly in its format?**

```text

```

**What type of database should you consider using if the data you're going to be storing will reliably be in the same structured format?**

```text
relational database
```

**In our example, once a record of a book is inserted into our Books table, it would be represented as a ___ in that table?**

```text
row
```

**Which type of key provides a link from one table to another?**

```text
foreign key
```

**Which type of key ensures a record is unique within a table?**

```text
primary key
```

## 🧭 Task 3: SQL

### 📖 What is SQL?

SQL (Structured Query Language) is used to communicate with relational databases.

Instead of manually browsing data, SQL allows users to:

* Create databases
* Create tables
* Insert data
* Retrieve data
* Update data
* Delete data

### 🏢 What is a DBMS?

A Database Management System (DBMS) acts as the interface between users and databases.

Examples:

* MySQL
* MariaDB
* Oracle Database
* MongoDB

The DBMS processes SQL queries and manages the underlying database.

### 🚀 Benefits of SQL

#### ⚡ Fast

Efficiently processes large datasets.

#### 📖 Easy to Learn

Uses readable English-like syntax.

Example:

```sql
SELECT * FROM books;
```

#### 🎯 Reliable

Enforces data structure and consistency.

#### 🔍 Flexible

Provides powerful querying capabilities.

## 🧭 Task 4: Database and Table Statements

These statements are used to create and manage databases and tables.

### 🏗 CREATE DATABASE

Create a new database.

```sql
CREATE DATABASE thm_bookmarket_db;
```

### 📋 SHOW DATABASES

List all available databases.

```sql
SHOW DATABASES;
```

### 🎯 USE

Select the active database.

```sql
USE thm_bookmarket_db;
```

### ❌ DROP DATABASE

Delete a database.

```sql
DROP DATABASE database_name;
```

### 📦 CREATE TABLE

Create a table.

```sql
CREATE TABLE book_inventory (
    book_id INT AUTO_INCREMENT PRIMARY KEY,
    book_name VARCHAR(255) NOT NULL,
    publication_date DATE
);
```

### 📑 SHOW TABLES

List tables in the active database.

```sql
SHOW TABLES;
```

### 🔍 DESCRIBE

Display table structure.

```sql
DESCRIBE book_inventory;
```

### ✏️ ALTER TABLE

Modify an existing table.

Example:

```sql
ALTER TABLE book_inventory
ADD page_count INT;
```

### 🗑 DROP TABLE

Delete a table.

```sql
DROP TABLE table_name;
```

## 🧭 Task 5: CRUD Operations

CRUD represents the four core database operations.

### ➕ Create (INSERT)

Adds new records.

```sql
INSERT INTO books
(id, name, published_date, description)
VALUES
(1, "Android Security Internals",
"2014-10-14",
"An In-Depth Guide to Android's Security Architecture");
```

### 📖 Read (SELECT)

Retrieve data.

All columns:

```sql
SELECT * FROM books;
```

Specific columns:

```sql
SELECT name, description FROM books;
```

### ✏️ Update (UPDATE)

Modify existing records.

```sql
UPDATE books
SET description = "Updated Description"
WHERE id = 1;
```

### 🗑 Delete (DELETE)

Remove records.

```sql
DELETE FROM books
WHERE id = 1;
```

### CRUD Summary

| Operation | SQL Statement |
| --------- | ------------- |
| Create    | INSERT        |
| Read      | SELECT        |
| Update    | UPDATE        |
| Delete    | DELETE        |

## 🧭 Task 6: Clauses

Clauses define how data should be filtered, grouped, or sorted.

### DISTINCT

Removes duplicate values.

```sql
SELECT DISTINCT name
FROM books;
```

### GROUP BY

Groups rows together.

```sql
SELECT name, COUNT(*)
FROM books
GROUP BY name;
```

### ORDER BY

Sorts results.

Ascending:

```sql
ORDER BY published_date ASC;
```

Descending:

```sql
ORDER BY published_date DESC;
```

### HAVING

Filters grouped results.

```sql
SELECT name, COUNT(*)
FROM books
GROUP BY name
HAVING name LIKE '%Hack%';
```

## 🧭 Task 7: Operators

Operators allow filtering and comparison of data.

### 🔍 LIKE

Pattern matching.

```sql
WHERE description LIKE "%guide%"
```

### ➕ AND

All conditions must be true.

```sql
WHERE category = "Offensive Security"
AND name = "Bug Bounty Bootcamp"
```

### 🔀 OR

At least one condition must be true.

```sql
WHERE name LIKE "%Android%"
OR name LIKE "%iOS%"
```

### 🚫 NOT

Negates a condition.

```sql
WHERE NOT description LIKE "%guide%"
```

### 📏 BETWEEN

Checks ranges.

```sql
WHERE id BETWEEN 2 AND 4
```

### ⚖️ Comparison Operators

| Operator | Meaning               |
| -------- | --------------------- |
| =        | Equal                 |
| !=       | Not Equal             |
| <        | Less Than             |
| >        | Greater Than          |
| <=       | Less Than or Equal    |
| >=       | Greater Than or Equal |

#### Equal

```sql
WHERE name = "Designing Secure Software"
```

#### Not Equal

```sql
WHERE category != "Offensive Security"
```

#### Less Than

```sql
WHERE published_date < "2020-01-01"
```

#### Greater Than

```sql
WHERE published_date > "2020-01-01"
```

#### Less Than or Equal

```sql
WHERE published_date <= "2021-11-15"
```

#### Greater Than or Equal

```sql
WHERE published_date >= "2021-11-02"
```

## 🧭 Task 8: Functions

Functions help manipulate and aggregate data efficiently.

### 🔤 String Functions

#### CONCAT()

Combines strings.

```sql
SELECT CONCAT(
    name,
    " is a type of ",
    category,
    " book."
)
FROM books;
```

#### GROUP_CONCAT()

Combines values from multiple rows.

```sql
SELECT category,
GROUP_CONCAT(name SEPARATOR ", ")
FROM books
GROUP BY category;
```

#### SUBSTRING()

Extracts part of a string.

```sql
SELECT SUBSTRING(published_date,1,4)
FROM books;
```

Output:

```text
2014
2021
2016
...
```

#### LENGTH()

Returns character count.

```sql
SELECT LENGTH(name)
FROM books;
```

### 📊 Aggregate Functions

#### COUNT()

Counts rows.

```sql
SELECT COUNT(*)
FROM books;
```

#### SUM()

Adds values.

```sql
SELECT SUM(price)
FROM books;
```

#### MAX()

Returns largest value.

```sql
SELECT MAX(published_date)
FROM books;
```

Output:

```text
2021-12-21
```

#### MIN()

Returns smallest value.

```sql
SELECT MIN(published_date)
FROM books;
```

Output:

```text
2014-10-14
```
