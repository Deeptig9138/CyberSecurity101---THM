# 📜 JavaScript Essentials

JavaScript (JS) is one of the most widely used scripting languages for web development. It adds interactivity and dynamic functionality to websites built with HTML and CSS.

JS enables features such as:

* Form validation
* Button click actions
* Dynamic page updates
* Animations
* User interaction
* Browser-based applications

From a cybersecurity perspective, understanding JavaScript is essential because attackers often abuse legitimate JS functionality to perform malicious actions such as Cross-Site Scripting (XSS), credential theft, browser manipulation, and client-side bypasses.

## 🧭 Task 2: Essential Concepts

JavaScript relies on a few core building blocks that developers use to create interactive applications.

### 📦 Variables

Variables act as containers that store data values.

Just as a bucket needs a label, variables require names so their contents can be referenced later.

### Variable Types

JavaScript provides three ways to declare variables:

```javascript
var
let
const
```

#### var

Function-scoped variable.

```javascript
var username = "THM";
```

#### let

Block-scoped variable.

```javascript
let age = 25;
```

#### const

Block-scoped constant that cannot be reassigned.

```javascript
const website = "TryHackMe";
```

### 🔤 Data Types

Data types define what kind of value a variable can store.

Common types include:

| Data Type | Example         |
| --------- | --------------- |
| String    | `"Hello"`       |
| Number    | `25`            |
| Boolean   | `true`          |
| Null      | `null`          |
| Undefined | `undefined`     |
| Object    | `{name:"John"}` |

### ⚙️ Functions

Functions group code that performs a specific task.

Example:

```javascript
function PrintResult(rollNum) {
    alert("Username with roll number " + rollNum + " has passed the exam");
}
```

Instead of writing the same code repeatedly, the function can be called whenever needed.

### 🔄 Loops

Loops execute code repeatedly while a condition remains true.

Common loop types:

* for
* while
* do...while

Example:

```javascript
for (let i = 0; i < 100; i++) {
    PrintResult(rollNumbers[i]);
}
```

This calls the function 100 times without manually writing it 100 times.

### 🌐 Request-Response Cycle

Web applications operate using a request-response model.

#### Request

The browser sends a request to a server.

#### Response

The server returns:

* Web pages
* Images
* Files
* Data

This communication forms the foundation of modern web applications.

### ✅ Answer

**What term allows you to run a code block multiple times as long as a condition is true?**

```text
loop
```

## 🧭 Task 4: Integrating JavaScript in HTML

JavaScript is typically used alongside HTML and CSS.

There are two primary integration methods:

1. Internal JavaScript
2. External JavaScript

### 📝 Internal JavaScript

Internal JS is written directly inside the HTML document.

Example:

```html
<!DOCTYPE html>
<html>
<body>

<p id="result"></p>

<script>
let x = 5;
let y = 10;
let result = x + y;

document.getElementById("result").innerHTML =
"The result is: " + result;
</script>

</body>
</html>
```

### How It Works

The script:

1. Calculates a value.
2. Finds an HTML element.
3. Updates the content dynamically.

### 🌐 External JavaScript

External JS stores code in a separate `.js` file.

#### script.js

```javascript
let x = 5;
let y = 10;
let result = x + y;

document.getElementById("result").innerHTML =
"The result is: " + result;
```

#### external.html

```html
<script src="script.js"></script>
```

The browser loads and executes the external file.

### 🔍 Identifying Internal vs External JS

View page source:

```text
Right Click → View Page Source
```

#### Internal JS

```html
<script>
...
</script>
```

#### External JS

```html
<script src="script.js"></script>
```

The `src` attribute indicates an external script.

## 🧭 Task 5: Abusing Dialogue Functions

JavaScript provides built-in dialogue functions that interact with users.

While useful, attackers can abuse them for social engineering and disruptive behavior.

### 🚨 Alert

Displays a message box.

```javascript
alert("Hello THM");
```

Result:

```text
[OK]
Hello THM
```

### ❓ Prompt

Requests user input.

```javascript
let name = prompt("What is your name?");
```

Example:

```javascript
alert("Hello " + name);
```

### ✅ Confirm

Provides a choice between OK and Cancel.

```javascript
confirm("Do you want to proceed?");
```

Returns:

```javascript
true
```

or

```javascript
false
```

### ⚠️ Malicious Example

```html
<script>
for (let i = 0; i < 3; i++) {
    alert("Hacked");
}
</script>
```

This repeatedly displays dialogue boxes.

A malicious actor could increase the loop count significantly and create a denial-of-service style annoyance.

## 🧭 Task 6: Bypassing Control Flow Statements

Control flow determines how a program executes based on conditions.

Common structures:

* if
* else
* switch
* for
* while
* do...while

### Conditional Statements

Example:

```javascript
age = prompt("What is your age");

if (age >= 18) {
    document.getElementById("message").innerHTML =
    "You are an adult.";
}
else {
    document.getElementById("message").innerHTML =
    "You are a minor.";
}
```

### 🔓 Client-Side Authentication Weakness

Developers sometimes implement login logic entirely in JavaScript.

Example:

```javascript
if(username == "admin" &&
   password == "ComplexPassword")
{
    // Login successful
}
```

Because JavaScript runs on the client side, attackers can inspect or modify the logic.

This demonstrates why authentication must always be enforced server-side.

## 🧭 Task 7: Exploring Minified Files

JavaScript source code is often transformed before deployment.

Two common techniques are:

* Minification
* Obfuscation

### 📦 Minification

Minification removes:

* Spaces
* Comments
* Line breaks

Example:

Before:

```javascript
function hi() {
    alert("Welcome to THM");
}
```

After:

```javascript
function hi(){alert("Welcome to THM")}
```

Functionality remains unchanged.

### 🔐 Obfuscation

Obfuscation intentionally makes code difficult to understand.

Example:

```javascript
function hi(){alert("Welcome to THM")}
```

May become:

```javascript
(function(_0x114713,_0x2246f2){...})();
```

The browser still executes it normally.

### Why Developers Use It

Benefits include:

* Smaller file size
* Faster page loads
* Increased difficulty for reverse engineering

### 🔍 Deobfuscation

Tools exist that can reverse many obfuscation techniques and restore readable code.

Therefore:

> Obfuscation slows attackers down but does not provide true security.

## 🧭 Task 8: Best Practices

When developing or assessing JavaScript applications, following secure coding practices is essential.

### 🛡 Avoid Client-Side Validation Only

Never trust client-side validation alone.

Attackers can:

* Disable JavaScript
* Modify requests
* Bypass controls

Always validate data on the server.

### 📚 Use Trusted Libraries

Avoid importing scripts from unknown sources.

Attackers often publish malicious packages that resemble legitimate libraries.

Always verify:

* Source
* Reputation
* Integrity

### 🔑 Avoid Hardcoded Secrets

Bad Practice:

```javascript
const privateAPIKey = "pk_TryHackMe-1337";
```

Users can view source code and recover exposed secrets.

Never store:

* API Keys
* Passwords
* Tokens
* Credentials

inside client-side JavaScript.

### 📦 Minify and Obfuscate Production Code

Benefits:

* Smaller files
* Faster loading
* More difficult reverse engineering

Remember:

> Obfuscation is not a security control, only a deterrent.
