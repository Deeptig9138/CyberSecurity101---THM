# 🔍 Gobuster: The Basics

Gobuster is one of the most widely used reconnaissance tools in penetration testing and bug bounty hunting. It helps security professionals discover hidden web directories, files, subdomains, and virtual hosts through brute-force enumeration.

From a cybersecurity perspective, Gobuster is an essential tool during the reconnaissance phase because it helps uncover hidden attack surfaces that may contain sensitive files, outdated applications, exposed admin panels, or vulnerable services.

## 🧭 Task 3: Gobuster Introduction

Gobuster is an open-source reconnaissance tool written in Go (Golang).

It performs brute-force enumeration against:

* Web directories
* Files
* DNS subdomains
* Virtual hosts
* Amazon S3 buckets
* Google Cloud Storage buckets

Gobuster is commonly used during:

```text
Reconnaissance → Scanning → Exploitation
```

### 📚 Enumeration

Enumeration is the process of identifying and listing resources.

Examples:

* Directories
* Files
* Subdomains
* Virtual hosts

Example:

```text
/admin
/uploads
/images
```

### 🔨 Brute Force

Brute forcing means trying every possible option until a match is found.

Gobuster performs brute force using wordlists.

Example wordlist:

```text
admin
backup
images
uploads
login
```

Gobuster appends each word to the target and checks whether it exists.

### ⚙️ Viewing Gobuster Help

Display the main help page:

```bash
gobuster --help
```

### 🏷 Common Global Flags

| Flag      | Description            |
| --------- | ---------------------- |
| `-t`      | Number of threads      |
| `-w`      | Wordlist path          |
| `--delay` | Delay between requests |
| `--debug` | Debug output           |
| `-o`      | Save output to file    |

### 🚀 Basic Example

```bash
gobuster dir \
-u "http://www.example.thm/" \
-w /usr/share/wordlists/dirb/small.txt \
-t 64
```

#### Breakdown

```bash
gobuster dir
```

Uses directory enumeration mode.

```bash
-u
```

Specifies target URL.

```bash
-w
```

Specifies wordlist.

```bash
-t 64
```

Uses 64 concurrent threads.

### ✅ Answers

**What flag do we use to specify the target URL?**

```text
-u
```

**What command do we use for subdomain enumeration?**

```text
dns
```

## 🧭 Task 4: Directory and File Enumeration

Directory enumeration helps identify hidden directories and files that are not directly linked from a website.

Examples:

```text
/admin
/uploads
/backup
/secret
```

Attackers frequently use directory enumeration to discover:

* Admin panels
* Configuration files
* Backups
* Source code
* Hidden content

### 📂 Common Website Structure

Example WordPress installation:

```text
wordpress/
├── wp-admin
├── wp-content
└── wp-includes
```

### ⚙️ Important dir Mode Flags

| Flag | Description                |
| ---- | -------------------------- |
| `-c` | Send cookies               |
| `-x` | Search specific extensions |
| `-H` | Add headers                |
| `-k` | Skip TLS verification      |
| `-n` | Hide status codes          |
| `-s` | Show specific status codes |
| `-b` | Blacklist status codes     |
| `-U` | Username                   |
| `-P` | Password                   |
| `-r` | Follow redirects           |

#### 🚀 Basic Directory Scan

```bash
gobuster dir \
-u "http://www.example.thm" \
-w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt
```

#### 🔄 Following Redirects

```bash
gobuster dir \
-u "http://www.example.thm" \
-w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt \
-r
```

#### 📄 Searching for Specific File Types

```bash
gobuster dir \
-u "http://www.example.thm" \
-w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt \
-x .php,.js
```

This searches for:

```text
index.php
login.php
app.js
main.js
```

### 🔍 Practical Example

Find hidden directories:

```bash
gobuster dir \
-u "http://www.offensivetools.thm" \
-w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt \
-r
```

Result:

```text
/secret
```

Enumerate inside the discovered directory:

```bash
gobuster dir \
-u "http://www.offensivetools.thm/secret" \
-w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt \
-x .js
```

### ✅ Answers

**Which flag skips TLS verification?**

```text
--no-tls-validation
```

**Which directory catches your attention?**

```text
secret
```

## 🧭 Task 5: Subdomain Enumeration

Subdomains are separate sections of a domain that may host different applications or services.

Examples:

```text
www.example.thm
shop.example.thm
blog.example.thm
academy.example.thm
```

Even if the main website is secure, a forgotten subdomain may still be vulnerable.

### ⚙️ Important dns Mode Flags

| Flag | Description         |
| ---- | ------------------- |
| `-d` | Target domain       |
| `-i` | Show resolved IPs   |
| `-c` | Show CNAME records  |
| `-r` | Custom DNS resolver |

### 🚀 Basic DNS Enumeration

```bash
gobuster dns \
-d example.thm \
-w /usr/share/wordlists/SecLists/Discovery/DNS/subdomains-top1million-5000.txt
```

#### How It Works

If the wordlist contains:

```text
www
shop
academy
```

Gobuster tests:

```text
www.example.thm
shop.example.thm
academy.example.thm
```

### Example Results

```text
Found: www.example.thm
Found: shop.example.thm
Found: academy.example.thm
Found: primary.example.thm
```

#### 🔍 Practical Example

```bash
gobuster dns \
-d offensivetools.thm \
-w /usr/share/wordlists/SecLists/Discovery/DNS/subdomains-top1million-5000.txt
```

### ✅ Answers

**Which shorthand flag specifies the target domain?**

```text
-d
```

**How many subdomains are configured for offensivetools.thm?**

```text
4
```

## 🧭 Task 6: Virtual Host Enumeration

Virtual Hosts (VHosts) allow multiple websites to run on a single server and IP address.

Example:

```text
www.example.thm
blog.example.thm
academy.example.thm
shop.example.thm
```

All may point to the same IP address.

### 🌐 Subdomains vs VHosts

#### Subdomains

Resolved through DNS.

```text
DNS → IP Address
```

#### VHosts

Determined using the HTTP Host header.

```text
Host: blog.example.thm
```

### ⚙️ Important vhost Flags

| Flag               | Description                 |
| ------------------ | --------------------------- |
| `-u`               | Target URL                  |
| `--append-domain`  | Append domain automatically |
| `--domain`         | Specify domain              |
| `--exclude-length` | Filter false positives      |
| `-r`               | Follow redirects            |
| `-m`               | HTTP method                 |

### 🚀 Basic VHost Enumeration

```bash
gobuster vhost \
-u "http://example.thm" \
-w /path/to/wordlist
```

#### Example Request

```http
GET / HTTP/1.1
Host: www.example.thm
```

Gobuster changes only:

```http
Host:
```

during enumeration.

Example:

```http
Host: blog.example.thm
Host: shop.example.thm
Host: academy.example.thm
```

### 🔍 Practical Example

```bash
gobuster vhost \
-u "http://10.48.143.106" \
--domain offensivetools.thm \
-w /usr/share/wordlists/SecLists/Discovery/DNS/subdomains-top1million-5000.txt \
--append-domain \
--exclude-length 250-320
```

#### Why Use --exclude-length?

Many invalid responses return:

```text
404
404
404
404
```

with identical page sizes.

Filtering by length removes these false positives.

### Example Valid Results

```text
blog.example.thm
shop.example.thm
academy.example.thm
www.example.thm
```

All returning:

```text
Status: 200
```

### ✅ Answer

**How many VHosts return HTTP 200 on offensivetools.thm?**

```text
4
```

## 📝 Key Takeaways

## 🚀 Common Command Cheat Sheet

### Directory Enumeration

```bash
gobuster dir -u TARGET_URL -w WORDLIST
```

### Search for Extensions

```bash
gobuster dir -u TARGET_URL -w WORDLIST -x .php,.js
```

### Subdomain Enumeration

```bash
gobuster dns -d DOMAIN -w WORDLIST
```

### VHost Enumeration

```bash
gobuster vhost -u TARGET_IP \
--domain DOMAIN \
-w WORDLIST \
--append-domain
```
